저번 포스팅에서 DXD9를 사용했는데 그 코드를 다시 DXD11로 변경하려 한다.
이미 win32 프로그램 자체가 오래된 방식이고, 또 학습용으로 사용하는 거라 오래된 api인 DXD9를 사용해도 상관없을 거라고 생각했는데,
DXD9와 DXD11에서 사용하는 용어들부터 큰 차이가 있어보이기 때문이다.

**[DXD9의 렌더링 과정]**
----초기화----
1.dx생성
2.dx Device 생성
----렌더링----
3.그릴 리소스 Device에 셋팅
4.((렌더링 함수들))
5.Device의 Present를 호출



**[DXD11의 렌더링 과정]**
----초기화----
1.DX Device, Immediate Context, Swap Chain 생성
2.Render Target View 생성
3.Render Target을 파이프라인에 바인드
4.Viewport 초기화
----렌더링----
5.그릴 리소스 Device에 셋팅
6.((렌더링 함수들))
7.Device의 Present를 호출


과정이 좀 바뀌었고, 복잡해졌다.

그림으로 보면 다음과 같다.





일단 DirectX에서 제공하는 튜토리얼을 보며 이 렌더링 과정을 하나씩 공부해볼 것이다.

```
#pragma comment(lib, "D3D11.lib")
#include <d3d11.h>
#include <D3DX11.h>
#include <D3DX10tex.h>
```
DXD9 대신 DXD11을 사용할 것이므로 헤더파일도 변경해준다.
DXD11은 아직 지원되는 라이브러리라 기본값으로 포함되어있는지 lib 파일을 명시하지 않고 헤더만 써도 빌드가 된다.

# 1. DX Device, Immediate Context, Swap Chain 생성

DX Device, Immediate Context, Swap Chain은 DXD11의 메인 루틴에서 가장 중요한 항목이다.

Direct 9에선 DX Device만으로 리소스를 생성하고 렌더링을 하고 최종 Draw까지 수행했지만, Direct 11에서는 리소스 생성과 렌더링이 분리되었다.
리소스 생성에 DX Device를, 렌더링에 Immediate Context를 사용한다.
Swap Chain은 버퍼를 관리하는 데에 쓰인다.


> ##### **DX Device**
Vertex 등 리소스를 생성하는 데 사용된다.


> ##### **Immediate Context**
DXD11에 추가된 항목이다.
Device가 생성한 리소스를 버퍼에 렌더링을 하는 데에 사용된다.

> ##### **Swap Chain**
Swap Chain은 렌더링을 할 버퍼를 얻는 데 필요하고, 컨텐츠를 실제 모니터에 보여주는 데에 쓰인다.
Swap Chain은 2개 이상의 버퍼를 갖는데 보통 Front Buffer와 Back Buffer이다. 이 버퍼들은 실제 모니터에 보여주기 위해 디바이스가 렌더링을 할 텍스쳐이다.
Front Buffer는 유저에게 현재 보여지는 버퍼로, 읽기전용이므로 변경할 수 없다.
Back Buffer는 Device가 앞으로 렌더링을 할 버퍼이다.
그리기 함수가 끝나면 Back Buffer가 Front Buffer가 되면서 유저에게 보여지고, Front Buffer는 Back Buffer가 되어 숨겨진다.
즉, 두 버퍼가 계속해서 역할 교체되며 화면을 보여주게 된다.

먼저 Swap Chain을 생성하려면 DXGI_SWAPCHAIN_DESC 구조체를 채워야 한다.

|필드|설명|
|---|---|
|**BackBufferUsage**| Back Buffer를 어떻게 사용할 것인지 설정한다. Back Buffer에 먼저 그리는 방식을 사용할 것이므로 DXGI_USAGE_RENDER_TARGET_OUTPUT으로 셋팅해준다.|
|**OutputWindow**| Swap Chain이 이미지들을 스크린에 보여주기 위해 사용할 윈도우를 뜻한다. 메인 윈도우의 핸들을 넣어주면 된다.|
|**SampleDesc**| Multi-sampling을 위해 사용된다. 사용하지 않으므로 1로 설정해주고, Quality는 0으로 설정해 비활성화한다.|
|**BufferDesc.Width, BufferDesc.Height**|버퍼의 크기를 정의한다. 해상도를 넣어준다.|

필드를 다 채우고나면 D3D10CreateDeviceAndSwapChain 함수를 호출해 Device와 SwapChain을 생성한다.

```
	//=================
	// Direct X 초기화
	//=================
	DXGI_SWAP_CHAIN_DESC swap_desc;
	ZeroMemory(&swap_desc, sizeof(swap_desc));
	swap_desc.BufferUsage = DXGI_USAGE_RENDER_TARGET_OUTPUT;
	swap_desc.BufferCount = 1;
	swap_desc.OutputWindow = hwnd_;
	swap_desc.SampleDesc.Count = 1;
	swap_desc.SampleDesc.Quality = 0;
	swap_desc.BufferDesc.Width = pt_resolution_.x;
	swap_desc.BufferDesc.Height = pt_resolution_.y;
	swap_desc.BufferDesc.Format = DXGI_FORMAT_R8G8B8A8_UNORM;
	swap_desc.BufferDesc.RefreshRate.Numerator = 60;
	swap_desc.BufferDesc.RefreshRate.Denominator = 1;
	swap_desc.Windowed = TRUE;

	IDXGISwapChain* p_swap_chain;
	if (FAILED(D3D11CreateDeviceAndSwapChain(NULL, D3D_DRIVER_TYPE_REFERENCE, NULL, 0, NULL, 0, 
		D3D10_SDK_VERSION, &swap_desc, &p_swap_chain, &p_d3d_device_, NULL, &p_immediate_context_))) {
		return E_FAIL;
	}
    
```

# 2. Render Target View 생성


> ##### **Render Target View란?**
DXD11에서 사용하는 Resource View의 타입 중 하나이다.
Resource View는 Resource가 파이프라인의 특정 단계에 바인드되도록 한다. 
C에서의 타입 캐스팅을 생각해보자. Raw 메모리 덩어리는 어떤 데이터 타입으로도 캐스팅될 수 있다. 하지만 실제 타입을 모른다면 사용할 수 없다.
DXD11의 Resource View도 비슷하게 동작한다. 예를들어 Raw 메모리 덩어리와 유사하게 Raw Resource가 있다.
이 Raw Resource를 가지고 있으면 이걸로 다른 Resource View를 만들고 여러 타입으로 그래픽 파이프라인의 여러 단계에 바인드할 수 있다. 이때의 타입이란, 렌더링할 Render Target 또는 Depth 정보를 받아올 Depth Stencil Buffer 등을 말한다.



Render Target View를 만들어야 한다. 그리고 이걸 Back Buffer에 Render Target으로 바인드해야 한다. DXD11은 이 과정을 거쳐야만 렌더링이 가능하다.
첫째로 GetBuffer()를 호출해 Back Buffer를 얻는다.
둘째로 CreateRenderTargetView()를 호출해 Render Target View를 생성한다. 두번째 인자로 D3D11_RENDERTARGETVIEW_DESC 구조체를 채워 Render Target View를 상세설정할 수도 있다. 여기선 불필요하므로 NULL을 넘겨 기본값으로 셋팅한다.


```
	// Render Target View 생성
	ID3D11Texture2D* p_back_buffer;
	if (FAILED(p_swap_chain_->GetBuffer(0, __uuidof(ID3D11Texture2D), (LPVOID*)&p_back_buffer))) {
		return E_FAIL;
	}

	if (FAILED(p_d3d_device_->CreateRenderTargetView(p_back_buffer, NULL, &p_render_target_view_))) {
		p_back_buffer->Release();
		return E_FAIL;
	}
	p_back_buffer->Release();

```

# 3. Render Target View를 파이프라인에 바인드



OMSetRenderTargets()를 호출해 Immediate Context를 파이프라인에 바인드한다. 이렇게 함으로써 파이프라인의 렌더링 출력이 Back Buffer에 쓰여지도록 한다.

```
	p_immediate_context_->OMSetRenderTargets(1, &p_render_target_view_, NULL);
```

# 4. Viewport 초기화


> ##### **Viewport의 역할**
Viewport는 Clip 공간 좌표를 Render Target 공간에 맵핑한다. X, Y좌표는 (-1, 1)까지, Z는(0, 1)이다.
Clip 공간이란 카메라에 보이지 않는 부분을 잘라내고 남은 공간을 말한다.

DXD9에서는 Viewport를 설정하지 않으면 기본값으로 Render Target과 같은 크기의 Viewport가 설정되었다. 하지만 DXD11에서는 자동으로 설정되지 않으므로 반드시 뭐라도 설정해주어야 한다.
Render Target 전체를 출력할 것이기 때문에 top left는 (0, 0)으로, width와 height는 Render Target 크기와 동일하게 설정해준다.


```
	// Viewport 초기화
	D3D11_VIEWPORT viewport;
	viewport.Width = pt_resolution_.x;
	viewport.Height = pt_resolution_.y;
	viewport.MinDepth = 0.0f;
	viewport.MaxDepth = 1.0f;
	viewport.TopLeftX = 0;
	viewport.TopLeftY = 0;
	p_immediate_context_->RSSetViewports(1, &viewport);
```


이제 초기화는 끝났다. DXD11를 사용해 네모를 그려볼 것이다.

# 5. 그릴 리소스 저장


> ##### **Vertex Buffer란?**
D3D11에서 네모를 바로 그릴 순 없고, 삼각형 두 개를 그려 네모를 만들어야 한다.
하나의 삼각형을 이루는 정보로는 좌표 세 개가 있다.
이런 정보들은 Buffer Resource에 저장되는데, 이 버퍼를 Vertex Buffer라고 부른다.


> ##### **Vertex 구조체란?**
Vertex는 위치값(좌표)과 색깔이나 텍스쳐 좌표를 갖는다.(텍스쳐 좌표는 텍스쳐 맵핑에만 사용된다.)
위치값만 가질 수도 있고, 위치값+@로 색깔이나 텍스쳐 좌표 등을 추가적으로 가질 수 있다.
즉, Vertex의 구조체는 정해져있지 않기 때문에 프로그래머가 직접 struct로 정의하여 사용한다.
그 장점으로 크기를 구할 때에도 편하게 sizeof(구조체)로 가져올 수가 있다.
대신에 GPU는 Vertex의 구조체 형식을 모르는 상태이기 때문에 형식을 알려줘야 한다.
이때 사용되는 게 Input Layout이다.


> ##### **Input Layout이란?**
Input Layout은 이 Vertex의 구조가 메모리에 어떤 순서로 저장되는지를 정의해, GPU가 값들을 해석할 수 있도록 한다.
각 Vertex의 구조는 D3D11_INPUT_ELEMENT_DESC 구조체로 정의한다.
Vertex 리스트를 정의하는 하나의 D3D11_INPUT_ELEMENT_DESC가 존재할 수도 있고, 각 Vertex에 대해 각각 다른 구조를 정의하기 위해 리스트를 사용할 수도 있다.
그리고 이 구조체는 최종적으로 하나의 Input Layout을 만드는 데에 쓰인다.



일단 Input Layout을 만들기 위해 D3D11_INPUT_ELEMENT_DESC를 설정해준다.

D3D11_INPUT_ELEMENT_DESC에 넣을 값들은 다음과 같은 것들이 있다.
|필드|설명|
|---|---|
|SegmanticName| 이 항목의 특성이나 목적을 설명하는 string 항목. 마음대로 이름지을 수 있다. POSITION, COLOR 같은 것들로 지을 수 있다.|
|SegmanticIndex| SegmanticName을 구분하기 위한 인덱스. SegmanticName이 COLOR일 때, 이 값이 0이면 COLOR0, 1이면 COLOR1 같은 식으로 같은 이름을 여러 개 사용하기 위해 쓰인다.|
|Format| 이 항목의 타입을 정의한다. 3개의 32bit 실수는 DXGI_FORMAT_R32G32B32_FLOAT으로 정의한다. 4개의 16bit unsigned 정수는 DXGI_FORMAT_R16G16B16A16_UINT로 정의한다.|
|InputSlot| D3D11에서는 Vertex 데이터를 GPU에게 넘기기 위해 Vertex Buffer를 사용한다. 이때 최대 16개의 Vertex Buffer를 사용할 수 있다. 0번 Slot부터 15번 Slot까지 바인드 가능하다. GPU에게 몇 번째 Vertex Buffer가 이 항목에 패치돼야하는지 알려준다.|
|AlignedByteOffset| Vertex Buffer에 저장된 Vertex의 시작 주소 offset을 지정한다.|
|InputSlotClass| 일반적으로 D3D11_INPUT_PER_VERTEX_DATA를 넣는다. 인스턴싱을 사용한다면 D3D11_INPUT_PER_INSTNACE_DATA를 사용할 수 있다.(인스턴싱: 같은 오브젝트 여러개를 배치할 때 병목현상을 방지하는 기법이라고 함)|
|InstanceDataStepRate| 인스턴싱을 위한 필드이다. 사용하지 않으므로 0을 넣는다.|


좌표를 저장하는 XMFLOAT3 POSITION과 XMFLOAT4 COLOR 두 가지 변수를 사용할 것이므로 이 둘을 넣어준다.

```
	D3D11_INPUT_ELEMENT_DESC layout[] =
	{
		{ "POSITION", 0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 0, D3D11_INPUT_PER_VERTEX_DATA, 0 },
		{ "COLOR", 0, DXGI_FORMAT_R32G32B32A32_FLOAT, 0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 }
	};
	UINT num_elements = ARRAYSIZE(layout);
```
여기서 POSITION과 COLOR는 각각 DXGI_FORMAT_R32G32B32_FLOAT과 DXGI_FORMAT_R32G32B32A32_FLOAT을 갖는다.
POSITION은 x, y, z 세 개의 float값을 갖고,
COLOR는 r, g, b, a 네 개의 float값을 갖는다는 뜻이다. (float 하나가 32bit)




여기서부터는 객체화 기술과 쉐이더의 영역이다.

Vertex Layout(=Input Layout)은 Vertex Shader와 밀접하게 연관되어 있다. Vertex Layout 객체를 만들려면 Vertex Shader의 Input Signature가 필요하기 때문이다.

따라서 Vertex Shader가 필요한데...
이 부분은 문서가 따로 분리되어 있다.

### 5-1. Vertex Shader

> ##### **Shader**
GPU에 의해 실행되는 작은 프로그램으로, 특정 Input Data를 받고 이를 처리해 파이프라인의 다음 단계로 넘기는 일을 한다.
D3D11에서 지원하는 Shader는 세 종류다. Vertex Shader, Geometry Shader, 그리고 Pixel Shader.

> ##### **Vertex Shader**
입력으로 Vertex를 받는다. Vertex Buffer를 통해 GPU로 넘겨진 모든 Vertex에 대해 1번씩 실행된다.

> ##### **Geometry Shader**
입력으로 Primitive를 받는다. GPU로 넘겨진 모든 Primitive에 대해 1번씩 실행된다.
Primitive는 점, 선, 삼각형일 수 있다.

> ##### **Pixel Shader**
입력으로 Pixel(때로는 Fragment라고도 불린다)을 받는다. 우리가 그리려는 Primitive의 각 Pixel에 대해 1번씩 실행된다.

D3D11에서 GPU는 반드시 유효한 Vertex Shader와 Pixel Shader를 갖고 있어야 한다. Geometry Shader는 선택적이다.(없어도 됨. 이 튜토리얼에선 사용되지 않음.)


**Vertex Shader**는 Vertex들에 대해 GPU가 실행하는 작은 프로그램이다.
Vertex들을 입력으로 받아 처리하고 변경해 출력하는 C 프로그램이라고 생각해보자.
이 프로그램이 Vertex 데이터를 GPU에게 Vertex Buffer를 통해 넘겨주고 난 뒤에는 GPU가 이 Vertex들을 쭉 읽어들인다.
그리고 각 Vertex에 대해 한 번씩 Vertex Shader를 실행한다.

Vertex Shader의 역할은 여러가지지만 그 중에서 가장 중요한 역할은 Transformation이다.
*Transformation*이란 어떤 좌표를 한 좌표계에서 다른 좌표계로 변환하는 것을 뜻한다. 예를 들어, 3D 씬에서의 삼각형은 아마 (0, 0, 0), (1, 0, 0), (0, 1, 0) 좌표를 가질 것이다.
이 삼각형이 2D Texture Buffer로 옮겨지면 GPU는 2D 좌표를 알아야 한다. 이때 Transformation이 이를 수행한다.
D3D11 튜토리얼에서 우리는 High-Level Shading Language(HLSL)을 사용할 것이다.
단순히 3D 좌표를 갖고 있는 Vertex 데이터를 불러들이기만 하는 Vertex Shader의 형태는 다음과 같다.

```

//--------------------------------------------------------------------------------------
// Vertex Input Type
//--------------------------------------------------------------------------------------
struct VertexInputType
{
    float4 position : POSITION;
    float4 color : COLOR;
};
//--------------------------------------------------------------------------------------
// Pixel Input Type
//--------------------------------------------------------------------------------------
struct PixelInputType
{
    float4 position : SV_POSITION;
    float4 color : COLOR;
};
//--------------------------------------------------------------------------------------
// Vertex Shader
//--------------------------------------------------------------------------------------
PixelInputType VS( VertexInputType input )
{
    PixelInputType output;
    output.position = input.position;
    output.color = input.color;
    return output;
}


```
C함수와 유사하게 생겼다. HLSL은 C 문법과 유사한 문법을 사용해서 C/C++ 프로그래머라면 익히기 쉽다.
이 코드에서 VS(Vertex Shader)가 파라미터로 float4 타입을 받고 리턴 형식도 float4 형식을 갖는 걸 볼 수 있다. HLSL에서 float4란 4개의 float-point 값을 갖는 벡터를 의미한다.
DX에서는 XMFLOAT4와 호환된다.
콜론은 파라미터의 시맨틱와 리턴값을 정의한다. HLSL의 시맨틱은 데이터의 특징을 설명한다. 아까 위에서 layout에 정의한 POSITION과 COLOR가 바로 시맨틱이다.
```
struct CustomVertex {
	XMFLOAT3 position;
	XMFLOAT4 color;
};
```
앞으로 그리기 함수를 호출하게 전에 버퍼에 Vertex를 채울 텐데, 이 Vertex는 VertexInputType의 struct와 호환되는 struct를 갖는다.
다만 VertexInputType의 position은 float4인데, CustomVertex의 position은 XMFLOAT3이다.
이것은 실제로 사용하는 좌표계는 3차원이지만 Vertex Shader에서 행렬 연산을 할 땐 필드 하나를 더 쓰기 때문으로 생각된다.
CustomVertex의 값을 Buffer에 채우면 VertexInputType으로 변환될 때 나머지 한 자리는 알아서 0으로 들어가기 때문에 신경쓰지 않아도 된다.



현대 컴퓨터 모니터는 래스터 디스플레이다. 스크린이란 게 픽셀이라 불리는 작은 점들로 이루어진 2차원 격자라는 뜻이다.
각 픽셀은 각각 다른 색깔 정보를 갖고 있다. 우리가 스크린에 삼각형을 그릴 때에, 실제로 삼각형을 그리는 게 아니다. 대신에 우리는 삼각형의 범위를 덮고 있는 모든 점들에 불을 켠다.

3개의 좌표로 정의된 삼각형을 해당 삼각형이 덮는 모든 픽셀의 덩어리로 변환하는 작업을 Rasterization(래스터화)이라고 부른다.
GPU는 첫째로 삼각형이 그려질 때 어떤 픽셀이 덮여지는지 결정한다. 둘째로 이 모든 픽셀에 대해 활성화된 **Pixel Shader**를 호출한다. Pixel Shader의 주 목적은 각 픽셀의 색깔 정보를 계산하는 것이다.
Shader는 색깔이 입혀져야 하는 픽셀을 입력받고 색깔을 계산한 뒤 파이프라인으로 출력한다. 이때 입력은 활성화된 Geometry Shader가 있을 경우 Geometry Shader로부터 받고, 만약 (튜토리얼처럼) 없을 경우 Vertex Shader로부터 직접 전달받는다.

우리가 위에서 정의한 Vertex Shader는  PixelInputType이란 시맨틱을 출력한다.
그런데 PixelInputType을 보면 SV_POSITION이라는, 아까 정의하지 않은 시맨틱이 보인다. 이건 미리 정의된 특별한 시맨틱이다. 그래픽 파이프라인에게 이 데이터가 클립 공간 위치와 관련이 있다는 걸 알려준다.
GPU가 픽셀을 스크린에 그리기 위해서는 SV_POSITION이 필요하다.

아래는 Pixel Shader의 코드이다.

```

//--------------------------------------------------------------------------------------
// Pixel Shader
//--------------------------------------------------------------------------------------
float4 PS(PixelInputType input) : SV_Target
{
    return input.color;
}
```

Pixel Shader는 SV_TARGET이라는 미리 정의된 시맨틱을 출력한다. 이 시맨틱은 Render Target 포맷을 나타낸다.



### 5-2. Shader들 생성하기
이제 Vertex Shader 오브젝트와 Pixel Shader 오브젝트를 만들 필요가 있다. 이 오브젝트들은 Shader들을 표현하고, D3DX11CompileFromFile()을 호출함으로써 생성된다.

Vertex Shader와 Pixel Shader를 메모장에 한 파일에 넣고, 이름을 "SimpleShader.hlsl"로 저장한다.
확장자는 .hlsl, .vs, .ps 등 여러가지를 쓰는 것으로 보아 어떤 확장자를 쓰든 상관 없는 것 같은데 난 .hlsl을 썼다.


이제 이 파일을 불러와 D3DX11CompileFromFile를 통해 Vertex Shader와 Pixel Shader의 데이터를 담은 ID3DBlob 객체를 얻는다.
여기서 Blob은 용량이 큰 데이터, 이미지나 동영상 사운드 같은 멀티미디어 데이터를 다룰 때 사용하는 Binary 객체이다.

ID3DBlob도 마찬가지로 binary data로, Vertex Shader의 Input Signature를 표현한다.

```
	//Vertex Shader 생성
	tstring shader_path = PathManager::GetInstance()->GetContentPath()+_T("shader\\SimpleShader.fx");
	ID3DBlob* p_vs_blob;
	ID3DBlob* p_vs_error_blob;
	if (FAILED(D3DCompileFromFile(shader_path.c_str(), NULL, NULL,
		"VS", "vs_4_0",

		D3DCOMPILE_ENABLE_STRICTNESS, 0, &p_vs_blob, &p_vs_error_blob)))
	{
		return E_FAIL;
	}
	ID3D11VertexShader* p_vs;
	p_d3d_device_->CreateVertexShader(p_vs_blob->GetBufferPointer(), p_vs_blob->GetBufferSize(), NULL, &p_vs);
	
	//Pixel Shader 생성
	ID3DBlob* p_ps_blob;
	ID3DBlob* p_ps_error_blob;
	if (FAILED(D3DCompileFromFile(shader_path.c_str(), NULL, NULL,
		"PS", "ps_4_0",
		D3DCOMPILE_ENABLE_STRICTNESS, 0, &p_ps_blob, &p_ps_error_blob)))
	{
		return E_FAIL;
	}
	ID3D11PixelShader* p_ps;
	p_d3d_device_->CreatePixelShader(p_ps_blob->GetBufferPointer(), p_ps_blob->GetBufferSize(), NULL, &p_ps);
```

이 과정을 거치면 Vertex Shader인 p_vs와 Pixel Shader인 p_ps가 얻어진다.


### 5-3. Input Layer(=Vertex Layout) 생성

이제 아까 정의한 layout으로 Input Layout을 생성하고 ID3D11DeviceContext::IASetInputLayout()을 호출해 활성 Vertex Layout으로 설정한다.
그리고 위에서 생성한 Shader들을 등록해준다.

```
	// Input Layout(Vertex Layout) 생성
	ID3D11InputLayout* p_input_layout;
	D3D11_INPUT_ELEMENT_DESC layout[] =
	{
		{ "POSITION", 0, DXGI_FORMAT_R32G32B32_FLOAT, 0, 0, D3D11_INPUT_PER_VERTEX_DATA, 0 },
		{ "COLOR", 0, DXGI_FORMAT_R32G32B32A32_FLOAT, 0, D3D11_APPEND_ALIGNED_ELEMENT, D3D11_INPUT_PER_VERTEX_DATA, 0 }
	};
	UINT num_elements = ARRAYSIZE(layout);
	if (FAILED(p_d3d_device_->CreateInputLayout(layout, num_elements, p_vs_blob->GetBufferPointer(),
		p_vs_blob->GetBufferSize(), &p_input_layout))) {
		return E_FAIL;
	}
	
	p_immediate_context_->IASetInputLayout(p_input_layout);
	p_immediate_context_->VSSetShader(p_vs, NULL, 0);
	p_immediate_context_->PSSetShader(p_ps, NULL, 0);
```


### 5-4. 삼각형 그리기

여기까진 초기화 작업이었고, 앞으로 렌더링에 들어가는 코드를 작성해볼 것이다.

지금까지 사용하던 좌표계는 왼쪽 위가 (0,0)이고 오른쪽 아래가 (해상도 width, 해상도 height)인 좌표계였다.
그런데 DX는 왼쪽 위가 (-1, 1)이고 오른쪽 아래가 (1, -1)인 좌표계를 사용한다.
모든 GObject의 position이 (0,0)~(width, height) 좌표계를 사용하고 있는데 DX에 출력할 땐 (-1, 1)~(1, -1)범위의 좌표계를 써야 한다.

스크린 좌표계 -> (변환 공식) -> DX 좌표계

이렇게 변환 공식을 거쳐 변환되도록 할 것이다.


##### 1. 변환 공식의 첫번째 단계는 스크린 좌표계를 (0,0)~(1,1) 좌표계로 변환하는 것이다.
이것은 DX에서 제공하는 함수
```
XMMATRIX projection = XMMatrixOrthographicLH(viewport.Width, viewport.Height, viewport.MinDepth, viewport.MaxDepth);
```
로 간단히 수행할 수 있다.
스크린 좌표에 projection을 곱하면 (0,0)~(1,1) 좌표계에서의 값으로 변환된다.

이 값을 Vertex Shader가 사용할 수 있도록 Constant Buffer를 생성해 그 안에 write 해준다.
```
	D3D11_BUFFER_DESC vs_const_buffer_desc;
	ZeroMemory(&vs_const_buffer_desc, sizeof(vs_const_buffer_desc));
	vs_const_buffer_desc.Usage = D3D11_USAGE_DYNAMIC;
	vs_const_buffer_desc.ByteWidth = ((sizeof(XMMATRIX)*2+15)/16)*16;
	vs_const_buffer_desc.BindFlags = D3D11_BIND_CONSTANT_BUFFER;
	vs_const_buffer_desc.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;
	vs_const_buffer_desc.MiscFlags = 0;
	vs_const_buffer_desc.StructureByteStride = 0;
	if (FAILED(p_d3d_device_->CreateBuffer(&vs_const_buffer_desc, NULL, &p_vs_const_buffer_)))
	{
		return E_FAIL;
	}
	//버퍼에 입력
	D3D11_MAPPED_SUBRESOURCE mapped_resource;
	if (FAILED(p_immediate_context_->Map(p_vs_const_buffer_, 0, D3D11_MAP_WRITE_DISCARD, 0, &mapped_resource)))
	{
		return E_FAIL;
	}
	VSConstBufferType* p_vs_const_data;
	p_vs_const_data = (VSConstBufferType*)mapped_resource.pData;
	p_vs_const_data->projection = projection;
	p_immediate_context_->Unmap(p_vs_const_buffer_, 0);
	p_immediate_context_->VSSetConstantBuffers(0, 1, &p_vs_const_buffer_);
```
```
struct VSConstBuffer {
	XMMATRIX projection;
};
```


이제 좌표계는 (0,0)~(1,1) 범위로 압축되었다.


---

##### ***\*Constant Buffer의 크기\****

여기서 Constant Buffer를 만들 때 계속 에러가 났는데,


문서를 찾아보니 ByteWidth를 16의 배수로 설정하라고 한다.
ByteWidth는 원래 버퍼의 크기, 그러니까 sizeof(VSConstBufferType)으로 잡는데 Constant Buffer는 특수한 경우인 것 같다.
16의 배수이면서 D3D11_REQ_CONSTANT_BUFFER_ELEMENT_COUNT보다 작게 설정하라고 하니,
MAX((((size+15)/16)*16), D3D11_REQ_CONSTANT_BUFFER_ELEMENT_COUNT)으로 잡으면 될 듯하다.

---

##### 2. 변환 공식의 두번째 단계는 y축을 반전시키고 (0, 0)의 위치를 왼쪽 위로 이동하는 것이다.
현재까지 변환된 결과를 보면, 화면의 왼쪽 위에 그려질 정점의 좌표는 (0, 0)을 갖고 있을 것이다.
하지만 이대로 DX의 그리기 함수를 사용하면 화면의 중앙 지점에 그려지게 된다.
DX의 좌표에서는 (0,0)이 중앙에 있고 (-1, 1)이 왼쪽 상단에 있기 때문이다.
즉, 내가 (0,0)에 그리고 싶다면 DX에서 그리기 전에 X축은 -1을 빼주고, Y축은 반전한 다음 1을 더해줘야 한다.
이 부분은 Vertex Shader에만 추가하면 된다.


완성된 Vertex Shader는 다음과 같다.

```
//SimpleShader.hlsl
//--------------------------------------------------------------------------------------
// GLOBAL
//--------------------------------------------------------------------------------------
cbuffer ParameterBufferType
{
    matrix projection;
};


//--------------------------------------------------------------------------------------
// Vertex Input Type
//--------------------------------------------------------------------------------------
struct VertexInputType
{
    float4 position : POSITION;
    float4 color : COLOR;
};

//--------------------------------------------------------------------------------------
// Pixel Input Type
//--------------------------------------------------------------------------------------
struct PixelInputType
{
    float4 position : SV_POSITION;
    float4 color : COLOR;
};


//--------------------------------------------------------------------------------------
// Vertex Shader
//--------------------------------------------------------------------------------------
PixelInputType VS( VertexInputType input )
{
    PixelInputType output;
    output.position = mul(input.position,projection);
    output.position.x = (-1.0 + output.position.x); //(-1, 1)의 x축을 (0, 1)로 변경
    output.position.y = (1.0 - output.position.y); //(1, -1)의 y축을 (0, 1)로 변경(x축 반전)
   
    output.color = input.color;
    return output;
}


//--------------------------------------------------------------------------------------
// Pixel Shader
//--------------------------------------------------------------------------------------
float4 PS(PixelInputType input) : SV_Target
{
    return input.color;
}

```




DX에서 삼각형을 그리는 방법은 다음과 같다.

(1) Vertex Buffer를 생성한다.
(2) 삼각형의 세 점을 Vertex Shader와 호환되는 구조체에 채운다.(여기서는 CustomVertex)
(3) Vertex Buffer에 (1)에서 채운 구조체를 write한다.
(4) 그리기 함수를 호출한다.

여기서 중요한 것은 Vertex Buffer를 얼마나 유지하고 있어야 하냐는 것이다.

Vertex Buffer는 GPU가 삼각형을 그리기 위한 좌표를 읽어올 버퍼이다.
게임 프로그램의 렌더링 함수는 최소한 1초에 60번은 실행된다.
Vertex Buffer를 매번 생성하고 삭제하는 건 권장하지 않는다고 하니, 그렇다면 하나의 삼각형은 매 렌더링에 같은 Vertex Buffer를 재활용해야 한다는 뜻이다.

검색해보니 하나의 텍스쳐엔 하나의 Vertex Buffer를 쓰는 게 정석이라고 한다.
텍스쳐란 Back Buffer 또는 Render Target View를 지칭하는 것 같다.

즉, 하나의 화면 안에 존재하는 모든 정점 데이터는 하나의 Vertex Buffer에 저장해야 한다.
(하나의 버퍼에 저장할 수 있는 정점은 128MB가 최대라고 한다.)

Vertex Buffer는 Draw 함수를 호출하기 전에 Vertex 정보를 임시로 저장하는 용도이기 때문에, 모든 캐릭터나 몬스터 등등 그림을 그려야 할 오브젝트들이 하나의 버퍼를 공유할 수 있다. 삼각형 100개를 그린다고 한다면 100개의 삼각형에 대한 정보를 버퍼 내에 계속 유지해야 하는 줄 알았는데, 그냥 당장 그릴 정보만 잠깐 가지고 있으면 되는 거였다.


그림으로 보면 대충 이런 느낌.




Vertex Buffer에는 두가지 종류가 있는데, 정적과 동적이다. 좌표값을 변경해야 한다면 동적 버퍼로 선언해야 한다. 동적 버퍼는 정적 버퍼보다 좀 느리다.
(게임 프로그램에서 정적 버퍼를 쓸 일이 뭐가 있을까 싶다. 수명주기동안 한 번도 좌표가 바뀌지 않는 데이터...)

하여간에 이제부터 동적 버퍼를 사용해서 사각형(=삼각형 두 개)을 그릴 것이다.


동적 버퍼를 만드는 부분은 초기화 부분에 추가해준다.


```
	//Vertex Buffer 생성
	D3D11_BUFFER_DESC buffer_desc;
	ZeroMemory(&buffer_desc, sizeof(buffer_desc));
	buffer_desc.Usage = D3D11_USAGE_DYNAMIC;
	buffer_desc.ByteWidth = sizeof(CustomVertex) * 65536;
	buffer_desc.BindFlags = D3D11_BIND_VERTEX_BUFFER;
	buffer_desc.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;
	buffer_desc.MiscFlags = 0;
	buffer_desc.StructureByteStride = 0;

	if (FAILED(p_d3d_device_->CreateBuffer(&buffer_desc, NULL, &p_vertex_buffer_))) {
		return E_FAIL;
	};

	// Set vertex buffer
	UINT stride = sizeof(CustomVertex);
	UINT offset = 0;
	p_immediate_context_->IASetVertexBuffers(0, 1, &p_vertex_buffer_, &stride, &offset);

	//Index Buffer 생성
	D3D11_BUFFER_DESC idx_buffer_desc;
	ZeroMemory(&idx_buffer_desc, sizeof(idx_buffer_desc));
	idx_buffer_desc.Usage = D3D11_USAGE_DYNAMIC;
	idx_buffer_desc.ByteWidth = sizeof(float) * 65536;
	idx_buffer_desc.BindFlags = D3D11_BIND_INDEX_BUFFER;
	idx_buffer_desc.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;
	idx_buffer_desc.MiscFlags = 0;
	idx_buffer_desc.StructureByteStride = 0;

	if (FAILED(p_d3d_device_->CreateBuffer(&idx_buffer_desc, NULL, &p_idx_buffer_))) {
		return E_FAIL;
	}

	// Set index buffer
	idx_format = DXGI_FORMAT_R32_UINT;
	p_immediate_context_->IASetIndexBuffer(p_idx_buffer_, idx_format, 0);
```

idx_format은 DXGI_FORMAT 타입으로, p_immediate_context에서 index buffer를 받아올 때 필요하기 때문에 멤버변수로 저장해주고
다른 함수에서 호출할 수 있도록 한다.


그리고 이제 그리기 함수.

Vertex Buffer와 Index Buffer의 내용만 변경하면 여러 그림을 그릴 수 있다.
일단 DX9에서처럼 선을 그리는 함수와 사각형을 그리는 함수로 나누어서 구현해보았다.
```


void DrawRectangle(ID3D11Device* p_d3d_device, const Vector2& base_pos, const Vector2& scale, ARGB line_color)
{
	ID3D11DeviceContext* p_immediate_context;
	p_d3d_device->GetImmediateContext(&p_immediate_context);


	//점 좌표
	const int vertice_count = 4;
	CustomVertex vertices[] =
	{
		{ XMFLOAT3(base_pos.x,				base_pos.y,				0.f), ARGB_TO_XMFLOAT(line_color)},
		{ XMFLOAT3(base_pos.x + scale.x,	base_pos.y,				0.f), ARGB_TO_XMFLOAT(line_color)},
		{ XMFLOAT3(base_pos.x + scale.x,	base_pos.y + scale.y,	0.f), ARGB_TO_XMFLOAT(line_color)},
		{ XMFLOAT3(base_pos.x,				base_pos.y + scale.y,	0.f), ARGB_TO_XMFLOAT(line_color)}
	};
	//선 인덱스
	const int line_indices_count = 8;
	UINT line_indices[] = {
		0, 1, 1, 2, 2, 3, 3, 0
	};

	//==========================
	// 선 그리기
	//==========================
	//Vertex Buffer 채우기
	WriteVertexBuffer(p_immediate_context, vertices, vertice_count);
	//Line Index Buffer 채우기
	WriteIndexBuffer(p_immediate_context, line_indices, line_indices_count);
	// Set primitive topology
	p_immediate_context->IASetPrimitiveTopology(D3D11_PRIMITIVE_TOPOLOGY_LINELIST);
	p_immediate_context->DrawIndexed(line_indices_count, 0, 0);

}

void DrawRectangle(ID3D11Device* p_d3d_device, const Vector2& base_pos, const Vector2& scale, ARGB line_color, ARGB plane_color)
{
	//면->선 순서로 그림.
	ID3D11DeviceContext* p_immediate_context;
	p_d3d_device->GetImmediateContext(&p_immediate_context);

	//점 좌표
	const int vertice_count = 4;
	CustomVertex vertices[] =
	{
		{ XMFLOAT3(base_pos.x,				base_pos.y,				0.f), ARGB_TO_XMFLOAT(plane_color)},
		{ XMFLOAT3(base_pos.x + scale.x,	base_pos.y,				0.f), ARGB_TO_XMFLOAT(plane_color)},
		{ XMFLOAT3(base_pos.x + scale.x,	base_pos.y + scale.y,	0.f), ARGB_TO_XMFLOAT(plane_color)},
		{ XMFLOAT3(base_pos.x,				base_pos.y + scale.y,	0.f), ARGB_TO_XMFLOAT(plane_color)}
	};
	//선 인덱스
	const int line_indices_count = 8;
	UINT line_indices[] = {
		0, 1, 1, 2, 2, 3, 3, 0
	};
	//면 인덱스
	const int plane_indices_count = 6;
	UINT plane_indices[] = {
		0, 1, 2, 2, 3, 0
	};

	//=======================
	// 면 그리기
	//=======================
	// Vertex Buffer 채우기
	WriteVertexBuffer(p_immediate_context, vertices, vertice_count);
	// Plane Index Buffer 채우기
	WriteIndexBuffer(p_immediate_context, plane_indices, plane_indices_count);
	// Set primitive topology
	p_immediate_context->IASetPrimitiveTopology(D3D11_PRIMITIVE_TOPOLOGY_TRIANGLELIST);
	p_immediate_context->DrawIndexed(plane_indices_count, 0, 0);

	//==========================
	// 선 그리기
	//==========================

	// Vertex 색깔 변경(line_color)
	for (int i = 0; i < vertice_count; i++) {
		vertices[i].color = ARGB_TO_XMFLOAT(line_color);
	}
	// Vertex Buffer 채우기
	WriteVertexBuffer(p_immediate_context, vertices, vertice_count);
	// Indices 변경(line index)
	WriteIndexBuffer(p_immediate_context, line_indices, line_indices_count);
	// Set primitive topology
	p_immediate_context->IASetPrimitiveTopology(D3D11_PRIMITIVE_TOPOLOGY_LINELIST);
	p_immediate_context->DrawIndexed(line_indices_count, 0, 0);


}

void WriteVertexBuffer(ID3D11DeviceContext* p_immediate_context, CustomVertex* vertices, UINT vertex_count)
{
	ID3D11Buffer* p_vertex_buffer;
	UINT stride = sizeof(CustomVertex);
	UINT offset = 0;
	p_immediate_context->IAGetVertexBuffers(0, 1, &p_vertex_buffer, &stride, &offset);

	D3D11_MAPPED_SUBRESOURCE mapped_resource;
	if (FAILED(p_immediate_context->Map(p_vertex_buffer, 0, D3D11_MAP_WRITE_DISCARD, 0, &mapped_resource)))
	{
		return;
	}
	CustomVertex* vertex_data = (CustomVertex*)mapped_resource.pData;
	for (int i = 0; i < vertex_count; i++) {
		vertex_data[i] = vertices[i];
	}
	p_immediate_context->Unmap(p_vertex_buffer, 0);
}

void WriteIndexBuffer(ID3D11DeviceContext* p_immediate_context, UINT* indices, UINT index_count)
{
	ID3D11Buffer* p_idx_buffer;
	DXGI_FORMAT idx_format = Core::GetInstance()->get_idx_format();
	p_immediate_context->IAGetIndexBuffer(&p_idx_buffer, &idx_format, 0);
	D3D11_MAPPED_SUBRESOURCE idx_mapped_resource;

	if (p_idx_buffer == NULL || FAILED(p_immediate_context->Map(p_idx_buffer, 0, D3D11_MAP_WRITE_DISCARD, 0, &idx_mapped_resource)))
	{
		return;
	}
	UINT* idx_data = (UINT*)idx_mapped_resource.pData;
	for (int i = 0; i < index_count; i++) {
		idx_data[i] = indices[i];
	}
	p_immediate_context->Unmap(p_idx_buffer, 0);
}

XMFLOAT4 ARGB_TO_XMFLOAT(ARGB argb)
{
	return XMFLOAT4(argb.r / 255.f, argb.g / 255.f, argb.b / 255.f, argb.a / 255.f);

}

```

ARGB_TO_XMFLOAT은 함수로 정의해준 것인데,
XMFLOAT4는 4byte float 4개로 a, r, g, b를 각각 표현한다.
이 프로그램에서는 색깔을 다루기 위해 RGBA라는 구조체를 새로 정의해서 0~255까지 범위의 a, r, g, b를 저장할 수 있게 하고,
그림을 그려줄 때에만 ARGB -> XMFLAOT4로 변환할 것이다.

```
union ARGB {
	struct {
		BYTE a;
		BYTE r;
		BYTE g;
		BYTE b;
	};

	ARGB(DWORD _argb) {
		b = _argb & 0xFF;
		g = _argb >> 8 & 0xFF;
		r = _argb >> 16 & 0xFF;
		a = _argb >> 24 & 0xFF;
	};
};
```
