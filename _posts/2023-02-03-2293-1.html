왜 한 번에 풀질 못할까...


접근하기

이 문제는 배낭문제 ver.2다.

특정 합이 되는 동전의 경우의 수를 구하는 문제.

시간제한이 무려 0.5초다. 브루트포스로 풀면 시간초과가 난다는 소리다.
거기다 메모리 제한이 4MB라는 건 직전 포스팅처럼 2차원 배열로 풀지 말고 1차원 배열로 풀라는 소리겠지.

저번 배낭문제와 아주 유사한데 이 문제는 무게라는 개념이 없다는 점이 다르다.
그리고 무조건 최대가치를 얻어내야 하는 배낭문제와 달리 '모든 경우의 수'를 세어야 한다.
또한, 동전은 한 종류를 여러 개 사용할 수 있다.
이것 때문에 풀이에서 한가지 큰 차이가 발생하는데, 그건 좀 이따가 설명하도록 하자.

일단 기본적인 포맷은 동일하다.
모든 아이템(동전)에 대하여 1부터 n번째 아이템까지 순회하면서, 모든 가치 1부터 v까지를 달성할 수 있는 경우의 수를 기록한다.

여기서 좀 헷갈린 게, 배낭의 경우 공간이 10인데 8만 채운 게 최적의 경우라면 그냥 그 값을 저장하는 게 최선이지만, 경우의 수는

- 이 동전을 선택해서 k를 만들 수 없는 경우 0으로 저장하면, 어떻게 다음번 순회 때 얼만큼의 공간이 남았는지를 알고 k를 만들 수 있는지 체크하지?
- 같은 동전을 여러 개 선택하는 건 어떻게 처리하지?

같은 의문들 때문에 삽질을 했다.
결론적으로 저런 의문들은 다 할 필요 없는 것들이었다.
어차피 0부터 시작해서 저장하기 때문에 앞에 저장된 값은 남지도 적지도 않은 정확히 k를 맞춘 경우의 수를 저장한다고 보장할 수 있기 때문이다.

이게 머리에 정확히 이해되진 않지만 그냥 개념적으로 기억해두자.
동적계획법에서 앞의 배열은 무조건 '완성된 상태'다.


1원, 2원, 5원짜리 동전이 존재할 때,
10원을 만들어야 한다고 생각해보자.

2원으로 10원을 만드는 경우의 수는,
[1] 2원 없이 다른 동전들로 8원을 만든 경우의 수
[2] 2원 없이 다른 동전들로 10원을 만든 경우의 수
를 더한 것과 같다.

2원을 선택하기 전에는, 표에 2원이 없는 상태로 8원을 만드는 경우의 수가 기록됐을 것이기 때문이다.
그리고 이제 현재 2원을 선택했으므로, 표에 기록된 8원을 만드는 경우의 수가 곧 10원을 만드는 경우의 수와 동일해진다.

코드로 표현하면
경우의 수를 저장한 배열 P가 있을 때,

동전 2를 선택한 순회에서,
P[10] = P[8] + P[10]
이라고 볼 수 있는 것이다.

이전에 어떤 동전을 선택했었는지, 이후에 어떤 동전을 선택할 것인지는 신경쓰지 않아도 된다.
매 순회에 최선의 선택을 기록하고 있기 때문이다.

2원이 없는 상태로 1원~10원을 만드는 경우의 수를 기록한 뒤,
2원을 선택한 상태로 1원~10원을 만드는 경우의 수를 더해 업데이트하면
(이전의 동전들+2원)으로 선택한 경우의 수를 저장하게 되는 것이다.

#include <iostream>
int P[10001]; //경우의 수를 저장
int coins[101];
int main() {
    int n, k;
    std::cin >> n >> k;
 
    for (int i = 1; i <= n; i++) {
        std::cin >> coins[i];
    }
 
    P[0] = 1;
    //n 종류의 동전으로 가치 l을 만들기 위한 최적의 경우의 수
    //가치 l을 만들기 위한 경우의 수는
    //가치 l-v을 만들기 위한 경우의 수 + 가치 l을 만들기 위한 경우의 수
    for (int i = 1; i <= n; i++) {
        for (int v = coins[i]; v <= k; v++) {
            P[v] = P[v - coins[i]] + P[v];
        }
    }
 
    std::cout << P[k];
 
 
 
 
}

의문점: P[0]=1인 이유

소스코드에서 볼 수 있듯, 여기서 P[0]의 초기값은 반드시 1이어야 한다.
그리고 P[1]~P[10]은 0이어야 한다.
개념적으로 순회를 돌기 전의 P[0]은 어떤 동전도 없는 상태에서 0원을 만드는 경우의 수인데, 그냥 생각하면 0인 게 맞을 것 같다. P[1]도, 어떤 동전도 없는 상태에서 1원을 만드는 건 불가능하니 0이다. P[10]까지도 다 0이다.
근데도 P[0]은 반드시 1로 저장해야 한다.

그 이유는, 새 동전을 선택했을 때, 순회의 첫 순서는 P[동전] = P[동전-동전] + P[동전]으로 시작하는데, (가령, 동전 2라면 P[2] = P[2-2]+P[2]부터 시작해서 P[10] = P[10-2] + P[10]까지 돌게 된다)
그렇기 때문에 P[동전]을 반드시 최초 1번 초기화하게 되는데,

이때
'2원짜리로 2를 만드는 경우의 수'는 반드시 1이어야 할 것이지만

매번 'v원짜리 동전으로 k원을 만드는 경우의 수'가 'v원 없이 k-2원을 만드는 경우의 수'와 동일하다고 간주하고 식을 세웠기 때문에, 
2원짜리로 2를 만드는 경우의 수가 곧 0원짜리로 2를 만드는 경우의 수가 된다.

2원짜리로 10원를 만드는 경우의 수 = 8원짜리로 10원을 만드는 경우의 수
둘이 같을 거라는 전제가,
선택한 동전과 만들려는 가격이 동일할 때에 한해서 문제가 생긴다는 거다.
즉, P[0]일 때 말이다.

P[1]부터는 문제가 없다.
2원짜리로 3원을 만들 때의 경우의 수는 당연히 P[1]+P[3]이다. P[1]이나 P[3]은 당장 0이어도 상관이 없다. 만약 1원짜리가 있다면 1원짜리를 순회할 때 1+기존값으로 갱신될 것이다. 문제는 여기서의 1도 P[0]이라는 점.
하지만 어쨌든 P[0]은 초기값이 0이라면 평생 갱신될 수 없다. 0원짜리는 존재하지 않기 때문이다.

이렇게 풀어쓰고 있지만 P[0]에서만 이런 문제가 발생하는 원리를 아직도 이해하지 못하겠다.

어쨌든 P[0]의 용도가 동전 n원짜리로 n원을 만들려는 경우의 수를 참조할 때에만 쓰인다는 점에 주목하자. 이외에는 쓰이지 않는다.
그래서 P[0]은 그냥 1로 초기화해버리면 된다.

0원을 만드는 경우의 수를 1로 저장해버리면 언젠가 나비효과로 연산 어딘가에서 문제가 생기는 거 아냐?! 라는 걱정이 들지만
실제로 그렇지 않다.
제시 문제에서 0원을 만드는 경우의 수도 케이스로 나오지 않는 한 상관없다.
만약 그렇다 해도 if(v==0) return 0 으로 처리해버리면 그만이다.

그러니 그냥 1로 저장하는 걸 외우자.


계속해서 P[0]=1 문제에 매몰되기 싫은데, 이걸 해결할 방법이 있을까 생각을 해봤다.
순회의 시작을 v=coins[i]가 아니라, v=coins[i]+1으로 치고,
순회를 시작하기 전에 P[coins[i]] = P[coins[i]]+1으로 미리 저장해버리면 어떨까?

int P[10001]; //경우의 수를 저장
int coins[101];
int main() {
    int n, k;
    std::cin >> n >> k;
 
    for (int i = 1; i <= n; i++) {
        std::cin >> coins[i];
    }

            P[v] = P[v - coins[i]] + P[v];
        }
    }
 
    std::cout << P[k];
 
}

의문점2 : 배열을 앞에서부터 저장하는 이유(동전의 중복이 미치는 영향)

이전 포스팅에서, 앞의 값을 참조해야 하기 때문에 뒤에서부터 저장해야 데이터의 손실을 막을 수 있다고 했었다.
오늘 이 문제 풀이를 보고 뭐지?싶었다.
앞에서부터 채워도 되는 거였나? 싶어 뒤에서부터 배열을 채웠더니 틀린다.
이 동전 문제도 앞의 값을 참조하기 때문에, 앞에서부터 배열을 채우면 다음 칸을 채울 때 갱신된 값을 참조해버리는 문제가 발생한다.
그런데도 반드시 앞에서부터 채워야 답이 맞고, 뒤에서부터 채우면 틀린다.

왜 이런 현상이 발생하는 것일까 생각을 해봤다.

문제는, '동전의 중복 여부'에 있었다.

앞서 배낭 문제는 넣을 수 있는 아이템이 종류당 1개씩이었다.
그래서 배열을 앞에서부터 채우면, 앞의 값을 참조했을 때 이미 해당 아이템이 반영된 최적의 해를 참조하게 된다.
가령 무게 10을 채우는데 현재 아이템 무게가 5라면, 그리고 무게5일 때 이 아이템을 선택한 게 최적의 해라면 그게 P[5]에 저장되어,
P[10]을 채울 때 P[5]를 참조하면 "이 아이템을 포함해 무게 5"인 상태의 최적의 해를 참조하게 된다. 그리고 또 이 아이템을 선택한 무게를 더해 P[10]에 저장해버릴 테니, 아이템을 여러 번 선택하게 된다.
 
그래서 한 아이템은 한 번만 선택하게 하기 위해 뒤에서부터 배열을 채워야 했던 것이다.

하지만 이 동전은 여러번 선택할 수 있기 때문에 애초에 앞에서부터 채우는 게 맞는 것이었던 것이다.


여기서 동적계획법 문제에서 아이템을 중복선택하느냐, 그렇지 않느냐에 따라 배열을 채우는 순서가 달라진다는 아주 중요한 사실을 알 수 있다.
즉, 위에서 중복처리를 신경쓰지 않아도 된다고 썼던 이유가 이것이다.
순회 순서만 뒤집어버리면 중복 여부를 선택할 수 있다.
