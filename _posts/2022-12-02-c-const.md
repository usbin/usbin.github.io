    ```
    class Object{
        
    public: 
        int size;
        int width;
        int height;
        const int& get_width() { return width;};
        const int& get_height() {return height;};
        const int& get_size(){ return int(get_width() + get_height()); };
    };
    
    ```
    멤버변수 내부에서 함수를 실행한 리턴값을 const &로 반환하는 이런 함수가 있다고 치자.
    
    
    ```
    int main()
    {
        Object* obj = new Object();
        obj->size = 0xFFFFFFFE;
        
        int s = obj->get_size();
        std::cout<<s;
        return 0;
    }
    ```
    이런 코드를 실행해버리면, get_size()가 실행되는 동안은 제대로 값이 들어가지만 main함수로 돌아오는 순간 해당 공간이 해제되어 변수 s는 해제된 공간을 참조하게 된다.
    
    const &로 값을 반환할 땐 참조로 반환하는 것이기 때문에 임시 값을 반환해선 안 되는데,
    컴파일 에러가 뜨면 좋겠지만 컴파일 할 땐 그냥 경고만 뜬다.
    
    그래서 문제가 왜 발생했는지 찾는 데에 한참 걸렸다.
    
    const &를 쓸 때는 반환한 뒤에 값이 사라지지 않는지 생각하고 써야한다.
