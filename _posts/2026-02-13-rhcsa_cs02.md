## 덤프 문제

```
YUM 기본 저장소를 아래와 같이 변경하세요.

아래와 같은 YUM 저장소가 사용가능합니다.

- http://content/rhel9.0/x86_64/dvd/BaseOS
- http://content/rhel9.0/x86_64/dvd/Appstream
```

## 풀이
```
# 1. 임의의 설정 파일을 생성
vi /etc/yum.repos.d/base.repo

# 2. /etc/yum.repos.d/base.repo 안에 작성
[BaseOS]
name=BaseOS Repository
baseurl=http://content/rhel9.0/x86_64/dvd/BaseOS
enabled=1
gpgcheck=0

[AppStream]
name=AppStream Repository
baseurl=http://content/rhel9.0/x86_64/dvd/Appstream
enabled=1
gpgcheck=0

# 3. :wq로 저장하고 나옴.

# 4. dnf 캐시 날리고 반영됐는지 확인
dnf clean all
dnf repolist

```

- \[ID]: 중복되지 않는 고유 ID. 대괄호 필수.
- name: 식별자
- baseurl: url 주소
- enabled: 활성화 여부
- gpgcheck: 0이면 서명 체크 하지 않음. 시험 환경에선 보통 0으로 설정한다고 함.
- 위의 항목들을 꼭 외워두자!!

## 여담

### 만약 dnf repolist에서 오류가 난다면??

네트워크(ip, gateway, dns) 설정이 제대로 안 잡혀있으면 오류가 난다고 하니 이쪽을 확인해볼 것.

### dnf repolist -v
진짜 저장소와 연결이 되는지 확인하려면 이 명령어로 확인하면 된다고 한다.
연결된 저장소들에서 metadata를 쫙 읽어오는 과정 같다.

근데 이게 덤프문제라 그런지 제시된 url로 접속도 되지 않았고, dnf repolist -v도 당연히 오류가 났다.
저장소 중 하나라도 막히면 metadata 받아오다가 멈춰버리니..
안 쓰는 저장소는 이름을 .repo로 끝나지 않게 바꿔버리거나
```bash
rename .repo .repo.bak *.repo
# 해석: *.repo에 해당하는 파일들 중에서 ".repo" 라는 글자를 찾아 전부 ".repo.bak"으로 바꿔라.
```
