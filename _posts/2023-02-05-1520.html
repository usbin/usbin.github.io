슬슬 동적계획법에 감이 잡히는 것 같다.


접근하기

이 문제도 점화식을 세워서 풀면 된다.
일단 DP[x][y]를 (1,1)에서 도착지까지 도착하기 위한 경우의 수라고 놓자.

DP[x][y] = DP[x-1][y] + DP[x+1][y] + DP[x][y-1] + DP[x][y+1]이 되는데,
상하좌우의 값이 다 필요하다.
여기서 "그럼 어디서부터 값을 채우기 시작해야 하는 거지?"란 의문이 들어서 굉장히 헷갈렸는데, 그냥 재귀로 구현하면 되더라.
Routes(x,y) = Routes(x-1, y) + Routes(x+1, y) + Routes(x, y-1) + Routes(x, y+1)
그리고 종료 지점을 구해야 하므로,
(1) x==1, y==1일 때는 1을 리턴하고,
(2) 이미 값이 저장됐을 경우엔 바로 그 값(DP[x][y])을 리턴한다.
이때 DP[x][y]에 이미 값이 저장된 상태인지를 구분하기 위해 초깃값을 전부 -1로 채워준다.
즉, DP[x][y]!=-1일 땐 return DP[x][y]를 바로 해주고,
(3) 값이 저장되지 않았을 경우, 즉 -1일 경우에만 점화식으로 다시 재귀호출을 해주면 된다.
Routes(x,y) = Routes(x-1, y) + Routes(x+1, y) + Routes(x, y-1) + Routes(x, y+1)

#include <iostream>
#include <memory.h>
int dx[] = {-1, 1, 0, 0};
int dy[] = {0, 0, -1, 1};
int max_y, max_x;
int map[501][501];
int DP[501][501]; //(1,1)->(i,j)로 가는 경로의 수
 
void Input(){
    std::cin>>max_y>>max_x;
    for(int i=1; i<=max_y; i++){
        for(int j=1; j<=max_x; j++){
            std::cin>>map[j][i];
        }
    }
}
int Routes(int x, int y){
    if(x==1 && y==1) return 1;
    if(DP[x][y] != -1) return DP[x][y];
    
    int max=0;
    for(int dir=0; dir<4; dir++){
        int next_x = x+dx[dir];
        int next_y = y+dy[dir];
        if(next_x>=1 && next_x<=max_x && next_y>=1 && next_y<=max_y){
            if(map[next_x][next_y]>map[x][y]){
                max += Routes(next_x,next_y);
            }
        }
    }
    DP[x][y] = max;
    return max;
}
void Solve(){    
    //방문하지 않음으로 설정.
    memset(DP, -1, sizeof(DP));
    
    std::cout<<Routes(max_x, max_y);
}
 
int main(){
    Input();
    Solve();
}

삽질의 시작: 스택 구현

여기서, 재귀로 구현할 수 있다는 건 스택 구현도 가능하다는 소리네? 란 생각이 들었는데,

정말 오랜 삽질 끝에 구현했다.

재귀는 간단한 데 반해 스택이 어려운 이유가 있다.

점화식 연산이 끝나는 시점이 제각각이라는 것이다.

어떤 칸 (x, y)가 있을 때, 재귀호출은 상하좌우의 DP값을 구할 때까지 쭉 올라간 다음에 구한 값을 가지고 내려와 DP[x, y]에 반영한다. 따라서 DP[x, y]의 값이 한 번에 초기화된다.

하지만 스택으로 구현할 경우, (x,y)를 스택에 넣은 다음, 상하좌우 DP값이 초기화되었다면 값을 더해주고 아직 초기화되지 않았으면 스택에 좌표를 넣어줘야 한다.
이때 주변 항목을 스택에 넣고 break를 해버린 후 나중에 주변항목의 초기화가 끝나고 이 항목으로 돌아왔을 때 덧셈을 마저 하게 된다. 문제는 그때 어느 값까지를 넣었는지 알 수 없다는 점에 있다...
재귀함수로 호출할 땐 함수 공간 자체가 스택이라 그 안에서 다른 함수를 몇 번을 호출하고 리턴값을 받아오든 해당 함수로 돌아오면 직전에 썼던 변수값이 유지되지만, 스택으로 구현하면 일일이 저장해줘야 한다.

처음에 이걸 어떻게 기억하게 할지 몰라서 한참 헤매다가,(대략 2시간...)
그냥 노드에 '어디까지 더했는가'를 같이 저장해주기로 했다.
더하는 순서는 dir을 0에서 3까지 증가시켜가며 더하므로, 항상 순서가 같다. 그래서 dir만 저장해줘도 다음에 스택에서 꺼냈을 때 중복없이 더할 수 있다.

struct Node {
    int x;
    int y;
    int dir;
};
 
 
int Routes(int x, int y) {
    DP[1][1] = 1;
    visited[1][1] = true;
    std::stack<Node> s;
    s.push(Node{ x, y, 0 }); // (x,y)에서 시작
 
    while (!s.empty()) {
        Node cnode = s.top();
 
        int all_searched = true;
        if (DP[cnode.x][cnode.y] == -1) DP[cnode.x][cnode.y] = 0;
 
        for (int dir = cnode.dir; dir < 4; dir++) {
            int next_x = cnode.x + dx[dir];
            int next_y = cnode.y + dy[dir];
            if (next_x >= 1 && next_x <= max_x && next_y >= 1 && next_y <= max_y) {
                //주변 항목에서 현재 항목으로 이동 가능한 경우
                if (map[cnode.x][cnode.y] < map[next_x][next_y]) {
                    //주변 항목의 값이 완성된 경우
                    if (visited[next_x][next_y]) {
                        DP[cnode.x][cnode.y] += DP[next_x][next_y];
                        s.pop();
                        s.push(Node{ cnode.x, cnode.y, dir+1 });
                    }
                    else{
                        all_searched = false;
                        s.push(Node{ next_x, next_y, 0});
                        break;
                    }
                }
            }
        }
        if (all_searched) {
            s.pop();
            visited[cnode.x][cnode.y] = true;
        }
    }
 
    return DP[x][y];
 
}

현재 노드에 주변항목의 DP를 더할 때마다 다음으로 더해야 할 다음 방향을 갱신해서 스택에 새로 저장해주었다. 스택에 넣었다 뺐다 하는 동작이 추가돼서 시간을 더 잡아먹지만, 이것보다 나은 방법을 모르겠다.
그리고 결과 시간이 똑같은 걸 보니 스택에서 삽입/삭제를 하는 건 O(1)이라 많이 해도 괜찮은듯.

그리고, 같은 이유로 visited라는 배열이 새로 하나 필요해진다.
재귀함수는 같은 시점에 끝나므로 한번에 참조하고 더하고 끝나버려서, DP가 -1이냐 아니냐만으로 판단할 수 있지만
스택은 각기 다른 시점에 끝나므로... DP를 여러차례 더하는 과정에 시간차가 발생한다. 따라서 -1이 아니어도 아직 더할 DP가 남아있다면 다음에 이 값을 만났을 때 값을 리턴하는 게 아니라 완성하기 위해 스택에 넣어야 할 수 있다.
따라서 visited라는 bool배열을 새로 선언해, 모든 값을 더했을 때에만 true가 되도록 설정해줬다.

#include <iostream>
#include <memory.h>
#include <stack>
int dx[] = { -1, 1, 0, 0 };

    int x;
    int y;
    int dir;
};
void Input() {
    std::cin >> max_y >> max_x;
    for (int i = 1; i <= max_y; i++) {
        for (int j = 1; j <= max_x; j++) {
            std::cin >> map[j][i];
        }
    }
}
int Routes(int x, int y) {

    visited[1][1] = true;

    s.push(Node{ x, y, 0 }); // (x,y)에서 시작
 
    while (!s.empty()) {
        Node cnode = s.top();
 
        int all_searched = true;
        if (DP[cnode.x][cnode.y] == -1) DP[cnode.x][cnode.y] = 0;
 
        for (int dir = cnode.dir; dir < 4; dir++) {
            int next_x = cnode.x + dx[dir];
            int next_y = cnode.y + dy[dir];
            if (next_x >= 1 && next_x <= max_x && next_y >= 1 && next_y <= max_y) {
                //주변 항목에서 현재 항목으로 이동 가능한 경우
                if (map[cnode.x][cnode.y] < map[next_x][next_y]) {
                    //주변 항목의 값이 완성된 경우
                    if (visited[next_x][next_y]) {
                        DP[cnode.x][cnode.y] += DP[next_x][next_y];
                        s.pop();
                        s.push(Node{ cnode.x, cnode.y, dir+1 });
                    }
                    else{
                        all_searched = false;
                        s.push(Node{ next_x, next_y, 0});
                        break;
                    }
                }
            }
        }
        if (all_searched) {
            s.pop();
            visited[cnode.x][cnode.y] = true;
        }
    }
 
    return DP[x][y];
 
}
void Solve() {

    memset(DP, -1, sizeof(DP));

}
