    # 1. 접근하기
    
    
    
    언뜻 보고 복잡한 줄 알았는데 뜯어보니 쉬운 문제였다.
    최단거리가 아니라 "원을 최대한 덜 거쳐서 도착하는" 조건이기 때문에, 빙빙 돌아가더라도 원만 안 거치면 된다.
    원을 불가피하게 거치게 되는 경우는 뭐가 있을까?
    1. 시작점이 원 안에 있는데 끝점은 원 밖에 있음
    2. 시작점이 원 밖에 있는데 끝점은 원 안에 있음
    두 가지밖에 없다.
    
    한마디로 시작점과 끝점 중 하나만 원 안에 있을 때, 이 원은 반드시 거쳐야 하는 원이 된다.
    
    이제 코드를 짜보자.

    반지름이 r이고 중심이 (a, b)인 어떤 원이 주어졌을 때 원 안에 있는지 밖에 있는지 판단하는 식은
    
    
    
    
    
    
    | | r^2 \>= (x-a)^2 + (y-b)^2 | | |--|--------------------------|--|
    
    
    
    
    
    
    
    이다.&nbsp;
    
    
    
    이 식을 적용해서 소스코드를 짜면 다음과 같다.
    
    # 2. 소스코드
    
    
    
    
    
    ```c++
    
    #include <iostream>
    #include <cmath>
    //점 (x, y)가 반지름이 r이고 중심이 cx, cy인 원 안에 있는가?
    bool isInCircle(int x, int y, int cx, int cy, int r){
        if(pow(r, 2) >= pow(x-cx, 2) + pow(y-cy, 2)){
            return true;
        }
        else return false;
    }
    int main(){
        //여러 원이 주어짐
        //시작점과 출발점이 주어짐
        //원을 최대한 덜 거쳐서 시작점-출발점을 지나도록 잇는다
        //몇 개의 원을 거치는가
        // -- 원 배열에서 시작점과 출발점 중 하나만 안에 있으면 count
        int T;
        std::cin>>T;
        // 테스트 케이스 T개
        for(int i=0; i<T; i++){
            int x1, y1, x2, y2, n;
            std::cin>>x1>>y1>>x2>>y2>>n;
            int count=0;
            //원 n개
            for(int i=0; i<n; i++){
                int cx, cy, r;
                std::cin>>cx>>cy>>r;
                if((isInCircle(x1, y1, cx, cy, r) && !isInCircle(x2, y2, cx, cy, r))
                   ||(!isInCircle(x1, y1, cx, cy, r) && isInCircle(x2, y2, cx, cy, r))){
                    count++;
                }
                
            }
            std::cout<<count<<"\n";
            
        }
    }
    
    ```
