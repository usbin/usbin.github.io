--- layout: post title: "[Unity] Unity를 github에 올릴 때 주의할 점" date: '2023-04-16T21:22:00.005-07:00' author: 가도 tags: - Unity modified\_time: '2023-04-16T21:22:34.220-07:00' blogger\_id: tag:blogger.com,1999:blog-778891705567453961.post-2939034800073673560 blogger\_orig\_url: https://sealbinary.blogspot.com/2023/04/unity-unity-github.html --- Unity 포폴을 만들며 git에 틈틈이 올리고 있었는데, 당연히 올라갔을 거라고 생각하고 진행하다가 거의 마무리할 쯤에야 용량 때문에 커밋만 되고 푸시가 되지 않고 있었다는 걸 깨달았다.

  

github엔 100mb 이상의 파일을 업로드할 수 없다. 100mb 이상의 파일을 업로드하려면 git lfs라는 대용량 스토리지를 따로 사용해야 한다.

코드만 작성해서는 하나의 파일이 100mb를 넘길 일이 없겠지만... 문제는 프로젝터 폴더 안에 자동으로 생성된 파일 중에 100mb를 넘는 파일이 있었다.

바로 ArtifactDB 파일.

대략 262mb를 차지한다.

  

100mb이상의 파일이 있으면 커밋만 되고 푸시는 되지 않는다.

그런데 한 번이라도 100mb 이상의 파일이 있는 상태에서 커밋을 했다면, 파일을 .gitignore에 포함시키고 캐시를 지우고 재커밋해도 이전의 커밋 기록에 100mb가 넘는 파일이 기록돼있기 때문에 푸시가 되지 않는다.

  

이런 경우 커밋 로그를 건드려야 푸시를 할 수 있었다.

  

[https://rtyley.github.io/bfg-repo-cleaner/](https://rtyley.github.io/bfg-repo-cleaner/)

  

이 사이트에서 프로그램을 다운받아 실행해주면 된다.(버전8 이상 자바가 깔려있어야 한다.)

그러면 커밋 로그가 수정되며 100mb 이상의 파일이 알아서 로그에서 지워진다.(실제로 삭제되는 것은 아님)

  

그런 뒤에 잊지말고 .gitignore에 대용량 파일을 포함시킨 뒤 커밋하고 푸시하면 된다.

  

그리고 이 프로그램을 쓰는 것보다 애초에 커밋할 때 100mb 이상의 파일이 있다는 경고를 발견하면 커밋을 취소하고 .gitignore로 제외시키는 게 훨씬 편하다.

  

아무튼 커밋할 땐 깃 로그를 꼭 확인하자.
