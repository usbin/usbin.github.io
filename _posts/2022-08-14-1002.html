백준 알고리즘 1002번(터렛) https://www.acmicpc.net/problem/1002




# 1. 접근하기
터렛1과 터렛2, 마린이 있을 때, 두 터렛의 좌표와 두 터렛에서 마린까지의 거리가 주어진다.
이때 마린이 있을 수 있는 좌표의 경우의 수의 개수를 구하는 문제다.
각 터렛에서 거리가 같은 점들을 이어놓고, 두 터렛의 범위가 겹치는 점의 개수를 구하면 된다.
쉽게 말해서, 두 원의 교점의 개수를 구하라는 소리다.
따라서 두 원이 있을 때 경우의 수를 나눠서 출력하면 된다.
그림으로 보면 이렇다.










1. 완전히 겹치거나
2. 한 점에서 만나거나
3. 두 점에서 만나거나
4. 만나지 않거나
네 가지 경우의 수는 두 점 사이의 거리와 반지름의 합, 차로 판별할 수 있다.


# 2. 소스코드

소스코드는 다음과 같다.

```c++
#include <iostream>
#include <cmath>
int main(){
    //좌표 (x1, y1)과 (x2, y2)가 주어졌을 때,
    //각 좌표와 마린까지의 거리 r1과 r2
    int T;
    int x1, y1, r1, x2, y2, r2;
    std::cin>>T;
    for(int i=0; i<T; i++){
        std::cin>>x1>>y1>>r1>>x2>>y2>>r2;
        //두 원이 교차하는 점의 개수를 찾기
        //무한대면 -1
        //경우의 수
        //(i) 정확히 겹침
        //x1==y1, x2==y2, r1==r2일 경우 무한대
        //(ii) 한 점에서 만남
        // - (1) 원의 밖에서 만남: 두 점 사이 거리가 반지름의 합과 같음
        // - (2) 원의 안에서 만남: 두 점 사이 거리가 반지름의 차와 같음
        //(iii) 두 점에서 만남
        // - 두 점 사이 거리가 반지름의 차보다 크고 합보다 작음
        //(iv) 안 만남
        // - (1) 두 점 사이 거리가 반지름의 차보다 작음
        // - (2) 두 점 사이 거리가 반지름의 합보다 큼
        double distance = sqrt(pow((x1-x2), 2) + pow((y1-y2), 2));
        double rAdd = r1+r2;
        double rSub = abs(r1-r2);
        if(x1==x2 && y1==y2 && r1==r2){
            std::cout<<-1;
        }
        else if(distance==rAdd || distance==rSub){
            std::cout<<1;
        }
        else if(distance>rSub && distance<rAdd){
            std::cout<<2;
        }
        else{
            std::cout<<0;
        }
        std::cout<<"\n";
        
        
    }
}
```
