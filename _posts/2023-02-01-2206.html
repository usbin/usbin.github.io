접근하기

지금 BFS문제만 풀고 있기 때문에 당연히 BFS 문제겠지만
코테에선 어떤 알고리즘을 써야 하는지 알 수 없기 때문에 알고리즘을 구별하는 능력을 길러야 한다.

이 문제는 최단경로를 찾는 문제이므로 BFS를 사용할 수 있다.
다만, '1회에 한정, 벽을 부술 수 있다'라는 선택지가 주어져 결국 모든 경로를 다 탐색해야 하므로 DFS로 풀어도 같은 효율을 보일 것으로 생각된다.

어쨌든 BFS로 풀어보자.


이 문제는 일반적인 BFS 문제처럼 '방문했다'라는 정보만 저장해선 풀 수 없더라.
모든 위치에 방문하는 데에 걸린 시간을 저장하고, 그 위치에 접근할 땐 저장된 값보다 작은 시간에 걸려서 도착했을 때에만 시간을 갱신하고 해당 위치에 도착해 탐색을 이어나가야 한다.

그 이유는 해당 위치에 가장 먼저 도착한다고 최단거리 해라는 보장이 없기 때문이다. 어떤 경로에서 먼저 X,Y라는 위치를 탐색한 다음에 또다른 경로가 해당 위치를 탐색할 때, 그 경로가 최단거리 해일 수도 있다.
이 차이는 '벽을 부순다'라는 선택지 때문에 생긴다.

만약 벽을 부술 수 없다면 모든 경로의 조건은 동일하기 때문에 특정 위치에 먼저 도착했다면 무조건 최단거리 해가 된다.

하지만 어떤 경로 A는 벽을 부수고 X,Y에 도착하고, 어떤 경로 B는 벽을 부수지 않고 X,Y에 도착했다면 당장은 어떤 해가 최단거리 해인지 알 수 없다.
A가 당장은 먼저 도착했어도 이후에 또 벽을 부술 차례가 되면 벽을 부술 수 없어 B보다 더 오래 걸리거나 아예 목적지에 도착하지 못할 수도 있다.

따라서 벽을 부순 A, 벽을 부수지 않은 B의 경우를 둘다 저장해야 한다.

최단거리 값의 갱신은 벽을 부순 경우끼리, 그리고 벽을 부수지 않은 경우끼리 경쟁한다.
매 탐색 때 벽을 만날 때마다 아직 벽을 한 번도 부수지 않았다면 '벽을 부순다'라는 선택을 하는 경우의 수를 큐에 추가한다.
벽을 부수지 않은 경우에서 최단거리를 저장하며 오다가 벽을 부순다는 선택을 하면, 그때부터 갱신은 벽을 부순 경우와 경쟁하는 것이다.

#include <iostream>
#include <queue>
#include <memory.h>
int g_count[1001][1001][2];
bool g_map[1001][1001];
 
const int dx[] = { -1, 1, 0, 0 };
const int dy[] = { 0, 0, -1, 1 };
struct Node {
    int x;
    int y;
    int count;
    bool break_wall;
};
void Bfs(Node node, int max_y, int max_x) {
    std::queue<Node> q;
    q.push(node);
    g_count[node.x][node.y][node.break_wall] = node.count;
 
    while (!q.empty()) {
        Node cnode = q.front();
        q.pop();
        
 
        for (int dir = 0; dir < 4; dir++) {
            Node new_node = cnode;
 
            new_node.x += dx[dir];
            new_node.y += dy[dir];
            new_node.count++;
            //g_count의 값을 갱신할 수 있는가
            if (new_node.x >= 1 && new_node.x <= max_x
                && new_node.y >= 1 && new_node.y <= max_y
                && (g_count[new_node.x][new_node.y][new_node.break_wall] > new_node.count
                    || g_count[new_node.x][new_node.y][new_node.break_wall] == -1)) {
 
                if (g_map[new_node.x][new_node.y]) {
                    //갈 수 있는 맵일 경우
                    q.push(new_node);
                    g_count[new_node.x][new_node.y][new_node.break_wall] = new_node.count;
                }
                else if (!new_node.break_wall
                    && (g_count[new_node.x][new_node.y][true] == -1
                    ||g_count[new_node.x][new_node.y][true]>new_node.count)) {
                    //벽이 있을 경우, 아직 벽을 부수지 않았을 경우, 그리고 부수고 넘어갔을 때 기존의 기록을 깰 수 있는 경우
                    new_node.break_wall = true;
                    q.push(new_node);
                    g_count[new_node.x][new_node.y][new_node.break_wall] = new_node.count;
                }
            }
        }
 
    }
    if (g_count[max_x][max_y][true] == -1
        && g_count[max_x][max_y][false] == -1) {
        std::cout << -1;
    }
    else if (g_count[max_x][max_y][true] == -1) {
        std::cout << g_count[max_x][max_y][false];
    }
    else if (g_count[max_x][max_y][false] == -1) {
        std::cout << g_count[max_x][max_y][true];
 
    }
    else {
        std::cout << std::min(g_count[max_x][max_y][true], g_count[max_x][max_y][false]);
    }
}
 
 
int main() {
    int N, M;
    std::cin >> N >> M;
    memset(g_count, -1, sizeof(g_count));
    for (int i = 1; i <= N; i++) {
        std::string s;
        std::cin >> s;
        for (int j = 1; j <= M; j++) {
            g_map[j][i] = s[j-1]=='0';
        }
    }
    //1,1에서 N,M까지 이동
    Node node{ 1, 1, 1, false };
    Bfs(node, N, M);
}

원래는 '방문여부'만 저장했던 2차원 배열에 해당 위치까지 도착할 수 있는 최단거리를 저장하면 다음과 같이 된다.

입력이 다음과 같을 때

모든 탐색을 마치면 g_count에는 다음과 같은 값이 저장된다.
(벽은 가독성을 높이기 위해 #으로 출력했다.)
위는 벽을 한 번도 뚫지 않고 해당 위치까지 걸리는 최단거리.
아래는 벽을 한 번 뚫은 상태로 해당 위치까지 걸리는 최단거리다.



이 테스트 케이스의 경우, 최단거리 해는 벽을 한 번도 뚫지 않고 위의 10번 자리까지 간 다음,
10번 아래 벽을 뚫으며 '벽을 뚫음'상태로 바뀌어 그 다음부턴 아래쪽 배열과 경쟁하게 된다. 따라서 위쪽엔(g_count[][][false] 배열) 벽을 뚫은 후부터의 값은 저장되지 않는다.

10번 아래 벽, 즉 아래쪽 배열의 6 아래 위치에서 다시 탐색을 시작하는데, 가독성을 높이기 위해 벽은 일관적으로 #으로 출력했지만 이 위치의 배열에 실제 저장된 값은 11로 갱신된 상태이다.

그리고 최종적으로 목적지인 맨 아래로 내려가며 거리값이 12가 된다.
