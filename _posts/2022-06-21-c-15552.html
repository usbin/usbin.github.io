지난주쯤 C#으로 이미 풀었던 문제.




얘는 문제에서 정답을 걍 다 알려준다.

하지만 실제 코테에서는 안 알려주고 본인이 외워서 써야 하니 함수를 통으로 외워두자.

#include <iostream> int main(){ std::cin.tie(NULL); //이 두 줄을 std::ios::sync_with_stdio(false); //외워야함 int T; std::cin>>T; for(int i=0; i<T; i++){ int A, B; std::cin>>A>>B; std::cout<<A+B<<"\n"; } }

따로 추가할 헤더는 없지만 네임스페이스가 2중이다.

sync_with_stdio가 std안의 ios안의 함수라는 걸 기억해두자.

​

추가로, 저 두 줄을 입력하는 것만으로 왜 실행속도가 올라가는가?

C++은 C의 입출력 스트림과 C++의 입출력 스트림을 동기화해서 하나로 사용하는데, std::ios::sync_with_stdio(false)를 사용하면 동기화를 중지한다. 대신 멀티쓰레드 환경에선 여러 스레드가 스트림에 접근하면서 문제가 발생할 수 있고, C의 입출력 함수를 사용할 수 없다.

C의 입출력 함수란

printf("abc"); scanf("%d", num); char a = getchar();

대충 이런 것들을 말한다.

​

std::cin.tie(NULL)은 입력 스트림을 출력 스트림으로부터 끊는 코드인데, 기본적으론 입력-출력이 묶여있어서 입력을 받으려 하면 자동으로 먼저 출력 스트림을 비워준다.

이 문제처럼 수만번의 입력을 받아야 할 때 입력을 받기 전에 매번 출력 스트림을 비워주면 실행속도가 느려지기 때문에 입출력 스트림을 끊어주면 속도가 빨라지는 것.

대신 출력 함수가 입력 함수 앞에 있어도 출력이 먼저 이루어진다고 보장할 수 없다는 단점이 있다.

​

물론 알고리즘 풀 땐 이 단점들을 딱히 신경 안 써도 된다.

다만 std::ios::sync_with_stdio(false) 사용했을 땐 C언어의 입출력 함수를 사용하지 않도록 주의하자.
