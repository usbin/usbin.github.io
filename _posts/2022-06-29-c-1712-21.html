이번 문제는 함수 문제다.

그냥 손으로 풀면 쉬운데 코드로 작성하려니까 헷갈렸다.

여기서 입력이 21억보다 작은 수라고 되어있는데, 곱셈을 하게 되면 당연히 이보다 커질 테니 안전빵으로 큰 자릿수에 저장했다.

int가 4바이트(-21억~21억 범위 저장)이니, 이보다 큰 자리수는 8바이트나 16바이트가 있을 것이다.

16바이트까진 필요없을 것 같고 여기서는 8바이트인 long long이나 __int64_t 타입을 사용하면 될 듯.

난 __int64_t를 사용했다. 참고로 여기서 64는 64비트라는 뜻이라서, 바이트로 환산하면 8바이트다.

#include <iostream> #include <cmath> int main(){ //A+B*n < C*n 인 지점 n을 찾기 __int64_t A, B, C; std::cin>>A>>B>>C; //C*n-B*n>A //n>A/(C-B) // A/(C-B) 보다 큰 정수 // k.0~k.9이면 -> k+1 int benefitPoint = floor((double)A/(C-B)+1); std::cout << (benefitPoint<0 ? -1 : benefitPoint); }

손익분기점을 구한 다음 그보다 "큰" 정수를 구해야 하므로, 결과에 1을 더해준 다음 내림 연산을 해줬다.
