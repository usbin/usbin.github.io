## 문제
```
아래와 같이 사용자/그룹 생성을 하라.

- _**sysmgrs**_라는 그룹을 생성하세요.
- _**natasha**_ 사용자를 생성하고, 보조 그룹으로 _**sysmgrs**_에 속하게 하세요.
- _**harry**_ 사용자를 생성하고, 보조 그룹으로 _**sysmgrs**_에 속하게 하세요.
- _**sarah**_ 사용자는 시스템에서 상호작용 shell에 접근할 수 없으며, _**sysmgrs**_ 그룹의 구성원이 아닙니다.
- _**natasha**_, _**harry**_, _**sarah**_의 비밀번호는 모두 _**flectrag**_로 설정하세요.
```

## 풀이
```
groupadd sysmgrs
useradd -G sysmgrs natasha
useradd -G sysmgrs harry
useradd -s /sbin/nologin sarah
echo "flectrag" | passwd --stdin natasha
echo "flectrag" | passwd --stdin harry
echo "flectrag" | passwd --stdin sarah

# 검증
id natasha
id harry
getent passwd sarah
```

## \<추가 정보>

### `-s /sbin/nologin`의 정체

이 옵션은 기본 쉘 경로를 /sbin/nologin으로 설정한다는 뜻인데, /sbin/nologin은 '쉘 로그인을 허용하지 않는 유저에게 설정하는 미리 약속된 경로'라고 한다.

`/etc/passwd` 파일에서 쉘 경로가 /sbin/nologin으로 설정돼있는 유저가 ssh 등의 원격이나 하여간 쉘 로그인을 시도하면 표준 메시지("This account is currently not available")을 띄우고 세션을 종료하게 된다.

시스템 내부에서 프로세스를 실행하기 위한 용도로만 존재하는 사용자계정은 쉘에 직접 로그인 할 필요가 없으므로 이렇게 막아두는 듯하다.
