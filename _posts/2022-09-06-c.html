# 1. 생성자 소멸자의 기본 동작

```c++
#include <iostream>

class Parent {
public:
    Parent() {
        std::cout << "Parent의 기본 생성자\n";
    }
    Parent(int n) {
        std::cout << "Parent의 int n 생성자\n";
    }
    ~Parent() {
        std::cout << "Parent의 소멸자\n";
    }
};

class Child : public Parent {
public:
    Child() {
        std::cout << "Child의 기본 생성자\n";
    }
    Child(int n) {
        std::cout << "Child의 int n 생성자\n";
    }
    ~Child() {
        std::cout << "Child의 소멸자\n";
    }
};

void ParentDefaultConstructor() {
    std::cout << "<-------- Parent p; ------------>\n";
    Parent p;
}

void ParentParamConstructor() {
    std::cout << "<-------- Parent p(1); ------------>\n";
    Parent p(1);
}

void ChildDefaultConstructor() {
    std::cout << "<-------- Child c; ------------>\n";
    Child c;
}

void ChildParamConstructor() {
    std::cout << "<-------- Child c(1); ------------>\n";
    Child c(1);
}
int main() {
    ParentDefaultConstructor();
    ParentParamConstructor();

    ChildDefaultConstructor();
    ChildParamConstructor();
}
    
```





부모 클래스의 경우 호출한 생성자가 그대로 호출이 되고,
자식 클래스의 경우 부모의 기본 생성자->본인의 호출한 생성자 순서로 호출이 된다.

여기서 Child c(1)을 호출했을 때 부모의 기본 생성자가 아닌 Parent(int n) 생성자를 호출하려면 멤버 이니셜라이저를 사용하면 된다.

# 2. 멤버 이니셜라이저

```c++
class Child : public Parent{

public:
	Child(int n) : Parent(n){ }

}
```
이렇게 생성자 뒤에 : Parent(인자)를 넣어주는 것이 멤버 이니셜라이저다.

이렇게 고쳐서 실행하면



부모도 int n 생성자가 실행된다.


# 3. 가상함수

가상함수란 부모의 함수 앞에 virtual 키워드를 붙여서 만든다.

```c++
class Parent{
	virtual void Func(){
   	//...
    }
}
```
가상함수는 포인터로 다룰 때 의미가 있는데,

```c++
Parent* pt = new Child();
delete pt;
```

이 경우, 결과는 다음과 같다.


부모의 타입으로 다루어도 생성은 Child형으로 되었기 때문에 생성자의 호출은 부모->자식 순차적으로 된다.
하지만 소멸자를 호출할 땐 Child의 소멸자가 분명 있는데도 호출되지 않는다.

이때 부모의 함수를 가상함수로 만들고 자식 함수에서 오버라이딩 해주면
```c++
#include <iostream>

class Parent {
public:
    Parent() {
        std::cout << "Parent의 기본 생성자\n";
    }
    Parent(int n) {
        std::cout << "Parent의 int n 생성자\n";
    }
    virtual ~Parent() { //virtual 키워드
        std::cout << "Parent의 소멸자\n";
    }
};

class Child : public Parent {
public:
    Child(){
        std::cout << "Child의 기본 생성자\n";
    }
    Child(int n) : Parent(n){
        std::cout << "Child의 int n 생성자\n";
    }
    ~Child() override { //override 키워드(생략 가능)
        std::cout << "Child의 소멸자\n";
    }
};
int main() {
    Parent* pt = new Child(1);
    delete pt;
}
```


Child의 소멸자가 먼저 실행되고 부모의 소멸자가 실행되게 된다.

virtual 키워드는 소멸자만이 아니라 부모 클래스의 모든 함수에 붙일 수 있다.

또한 클래스에 가상함수가 하나라도 있으면 가상함수 테이블이 생겨서 추가적으로 메모리를 잡아먹는다고 한다.

# 4. 순수 가상함수

순수 가상함수는
```c++
class Parent{
	virtual void Func()=0;
}
```
이렇게 만드는데, 순수 가상함수가 하나라도 있는 클래스는 단독으로 생성될 수 없고, 자식에게 상속하는 용도로만 쓸 수 있다.
또한 일반 가상함수는 자식 함수에서 오버라이딩하지 않아도 되지만 순수 가상함수는 반드시 오버라이딩해야 한다.
위의 Parent 클래스도 이제 
```c++
Parent p;
```
이렇게 단독으로 생성하려 하면 에러가 난다.


반드시 자식 함수에서 구현하게 하려 할 때, 혹은 부모 클래스를 단독으로 생성하지 못하게 할 때 사용한다.
