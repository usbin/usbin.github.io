말하자면 입력을 무한정 받는 코드.

사실 이러면 입력할 때까지 영원히 기다릴 것 같은데, 입력에 실패하면 cin.eof()가 false가 된다고 한다.

입력에 실패한다는 게 뭘까. 시간초과?

찾아보니 모든 파일의 마지막엔 EOF라는 표시가 붙어있다고 한다. 하지만 터미널에서 입력을 받을 땐 수동으로 넣어줘야 하는데, 윈도우에서 ctrl+Z, UNIX에서 ctrl+D가 EOF라고 함.

백준 알고리즘에서는 문제의 입력을 파일로 주기 때문에 EOF가 붙어있다고.

아무튼 소스코드는 아래와 같다.

#include <iostream> int main(){ int A, B; while(!(std::cin>>A>>B).eof()){ std::cout<<A+B<<"\n"; } }

​

처음에는

#include <iostream> int main(){ while(!std::cin.eof()){ int A, B; std::cin>>A>>B; std::cout<<A+B<<"\n"; } }

이렇게 풀어서 틀렸는데, 이게 왜 틀린가 찾아봤다.

알고보니 std::cin.eof()는 "읽기에 실패했을 때" false가 되는 것이기 때문에 std::cin에서 한 번이라도 먼저 읽기가 이루어져야 한다고 한다. 그럼 기본값이 true라는 뜻인가? 음...

같은 코드를 vs에서 돌리면 잘만 돌아가기 때문에... 100% 이해는 되지 않지만 어쨌든 외워두도록 하자.
