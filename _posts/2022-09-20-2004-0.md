    ### 1. 접근하기
    
    앞 문제 1676번과와 유사한데 식이 좀더 복잡해졌다.
    nCm을 구하라는데, nCm의 계산식은 n!/(n-m)!m!이다.
    
    앞 문제에서는 N!이었기 때문에 무조건 5가 2보다 많다고 보장할 수 있었지만 이번엔 나눗셈이 들어가기 때문에 2가 더 많을지 5가 더 많을지 모른다.
    따라서 더 적은 쪽을 선택해주는 부분이 추가돼야 한다.
    
    
    
    ### 2. 로직 세우기
    일단 n!/(n-m)!m!에 2가 몇 번 들어가는지는
    (n!에 곱해지는 2의 배수 개수) - ((n-m)!에 곱해지는 2의 배수 개수) - (m!에 곱해지는 2의 배수 개수)로 구할 수 있다.
    5도 마찬가지다.
    
    n!에 곱해지는 2의 배수 개수는 2가 몇 번 나오는지, 4가 몇 번 나오는지, 8이 몇 번 나오는지...를 세어서 더하면 되고
    (n-m)!와 m!도 마찬가지다.
    
    마지막으로 2와 5의 카운트 중 작은 값을 골라서 출력해주면 된다.
    
    ### 3. 소스코드
    ```c++
    #include <iostream>
    int main() {
        //n!/(n-m)!m!
    
        int n, m;
        std::cin >> n >> m;
        int nm = n - m;
    
        //n! 카운트
        int n_count_2 = 0;
        int n_count_5 = 0;
        for (long long i = 2; i <= n; i *= 2) {
            n_count_2 += n / i;
        }
        for (long long i = 5; i <= n; i *= 5) {
            n_count_5 += n / i;
        }
    
        //(n-m)! 카운트
        int nm_count_2 = 0;
        int nm_count_5 = 0;
        for (long long i = 2; i <= nm; i *= 2) {
            nm_count_2 += nm / i;
        }
        for (long long i = 5; i <= nm; i *= 5) {
            nm_count_5 += nm / i;
        }
    
        //m! 카운트
        int m_count_2 = 0;
        int m_count_5 = 0;
        for (long long i = 2; i <= m; i *= 2) {
            m_count_2 += m / i;
        }
        for (long long i = 5; i <= m; i *= 5) {
            m_count_5 += m / i;
        }
    
        int count_2 = n_count_2 - nm_count_2 - m_count_2;
        int count_5 = n_count_5 - nm_count_5 - m_count_5;
    
        std::cout << std::min(count_2, count_5);
    
    
    
    }
    ```
    
    
    여기서 n,m의 최댓값이 20억인데, int의 범위는 약 -21억~+21억이다.
    근데 for문의 인덱스로 쓰이는 i는 n*5보다 작은 값이 담길 수 있으므로, -105억~+105억의 값이 담길 수 있다.
    따라서 for문 변수엔 int보다 큰 자료형인 long long을 써줘야 한다.
