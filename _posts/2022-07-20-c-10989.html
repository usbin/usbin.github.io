1. 접근하기

이번 문제는 완전 창의력 문제였다.

접근법을 찾아보고나서야 이게 창의력 문제임을 깨달았음.

이 문제는 시간제한과 메모리제한이 걸려있다.

근데 수는 최대 10,000,000개를 입력받고, 이 수는 1부터 10,000까지의 자연수 중 랜덤으로 입력받게 된다.

int 자료형의 크기는 4byte.

10,000,000 x 4byte = 40MB

6MB 메모리로는 택도 없다.

그럼 어떻게 해야 하냐?

여기서 주목해야 하는 건, 수는 천만 개를 입력받지만 그 각각의 수는 1부터 10,000 사이에 있다는 거다.

입력받은 수를 저장하는 것보다 1부터 10,000까지의 배열을 만드는 게 더 메모리를 덜 먹는다.

10,000x4Byte = 40KB니까 메모리도 충분.

​

2. 로직 세우기

발상만 제대로 가면 로직 세우는 건 쉽다.

  1. 10000과 같거나 작은 수이기 때문에 10001칸짜리 배열을 만든다. 0으로 초기화한다.

  2. 수를 입력받고, 이 수를 인덱스로 해서 1번의 배열에서 ++ 연산을 한다.

  3. 모든 수를 입력받은 후, 0부터 10000까지 1번의 카운트 배열을 순회하며 저장된 수만큼 출력한다.

​

덧붙여, 시간제한이 있기 때문에 최대한 입출력을 빠르게 하기 위해 두 가지 코드를 추가해준다.

std::ios::sync_with_stdio(false); //c언어의 입출력 함수와 c++ 입출력 함수의 연동을 해제하여 더 빠른 연산 지원

입출력 속도를 개선할 때 쓰이는 코드로 std::cin.tie(NULL)도 있지만(입력과 출력의 연결을 끊음으로써 입력 전에 반드시 출력하지 않아도 되도록 하여 속도 개선)

이 코드는 입출력을 반복하지 않고 입력을 쭉 받고 다음에 출력을 쭉 받는 형태이기 때문에 std::cin.tie(NULL)은 추가하지 않았다.

3. 소스코드

#include <iostream> int main(){ std::ios::sync_with_stdio(false); int N; std::cin>>N; int numberArray[10001]; std::fill_n(numberArray, 10001, 0); for(int i=0; i<N; i++){ int input; std::cin>>input; numberArray[input]++; } for(int i=0; i<10001; i++){ if(numberArray[i] > 0){ for(int j=0; j<numberArray[i]; j++){ std::cout<<i<<"\n"; } } } }
