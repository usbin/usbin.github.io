### 1. 방향 벡터

이전에 미사일의 방향을 나타낼 때 벡터를 사용했다.
이때 벡터를 (0,1)이나 (1,0) 이런 식으로 넣었는데 매 프레임 벡터값\*속도\*dt만큼 이동하게 했다.

```c++
pos.x = direction_.x * move_speed_ * Time::GetInstance()->dt_f();
pos.y = direction_.y * move_speed_ * Time::GetInstance()->dt_f();
```
이러면 상하좌우로 움직일 땐 1 단위로 움직이지만 만약 대각선으로 움직인다면 1이 아니라 가로로1, 세로로1 이동하게 되어 실제로 루트2(약 1.414)만큼을 움직이게 된다.
대각선으로 움직일 때만 빨라진단 소리다.

의도한 건 상하좌우 대각선 어떤 방향이든 같은 속도로 날아가는 것이기 때문에 이걸 고쳐줄 것이다.

### 2. 벡터의 정규화

벡터의 정규화란 x와 y값을 대각선이 1일 때의 x와 y값으로 변경하는 것을 말한다.

x가 3, y가 4인 벡터가 있을 때, 이 벡터의 대각선 길이는 5이다.( ( 루트(3^2 + 4^2) ) = ( 루트(25) ) = 5 )
이 벡터를 정규화하면 x는 3/5, y는 4/5가 된다.
다른말로, 정규화된 벡터의 대각선은 항상 1이다.

즉 앞으로 상하좌우 대각선 어떤 각도로 이동하든 정규화된 벡터를 이용해 이동하면 1 단위로 움직일 수 있다는 뜻이다.

전에 선언한 vector2 구조체에 정규화 코드를 넣어준다.

```c++
	float Length() {
		return sqrtf(x * x + y * y);
	}
	Vector2& Normalize() {
		if (abs(x) < FLT_EPSILON && abs(y) < FLT_EPSILON) {
			return *this;
		}
		x /= Length();
		y /= Length();
		return *this;
	}
```
정규화는 x와 y를 각각 대각선 길이로 나누면 되는데 대각선 길이는 루트(x제곱 + y제곱)으로 구할 수 있다.

문제는 이때 x와 y가 0일 경우 대각선도 0이 되어 x와 y를 대각선으로 나눌 때 0으로 나누면서 예외가 발생해버린다.

따라서 x와 y가 0일 경우엔 정규화를 하지 않고 바로 (0, 0)인 벡터를 그대로 리턴해줘야 한다.

그리고 전에 포스팅했듯이 실수는 오차범위가 있기 때문에 정확히 0이 아니더라도 오차범위를 고려해 0이라고 간주해야 할 수 있다.
float 자료형의 오차범위인 FLT_EPSILON과 절댓값을 비교하면 0이라고 간주할 수 있는 값인지 확인이 가능하다.



### 3. 플레이어 이동 코드에 적용

플레이어가 이동할 때 정규화된 벡터를 사용하도록 바꾼다.

```c++
void Player::Update()
{
	Vector2 v = get_pos();

	Vector2 direction{ 0, 0 };
	if (KEY_HOLD(KEY::RIGHT)) direction.x = 1;
	else if (KEY_HOLD(KEY::LEFT)) direction.x = -1;
	if (KEY_HOLD(KEY::UP)) direction.y = -1;
	else if (KEY_HOLD(KEY::DOWN)) direction.y = 1;

	v.x += static_cast<float>(direction.Normalize().x) * speed_ * Time::GetInstance()->dt_f();
	v.y += static_cast<float>(direction.Normalize().y) * speed_ * Time::GetInstance()->dt_f();
	
	set_pos(v);

}
```
