이 문제는 처음 접근할 때 xxx666yyy...의 형태로 생각해서 저 666을 일의 자리에서 점점 자릿수를 높여가며 카운트를 세는 걸 생각했었는데, 중복수를 고려하지 못해서 틀렸다.

666666이라는 수가 있을 때, 이 수는 xxx666에서도 집계가 되고 xx666y, x666yy, 666yyy 모두에서 집계가 돼버린다.

그래서 그냥 숫자 0부터 666이 들어가는 수를 세면서 N에 도달하면 리턴하는 걸로 했다.

좀 무식한 접근방법 아닌가 싶지만 브루트포스 알고리즘에 속해있기 때문에...

​

#include \<iostream\>#include \<string\>#include \<time.h\>int main() { int N; std::cin \>\> N;//N번째 영화의 제목을 찾기.//N번째 영화는 666이 들어간 형태임 int number = 0;while (N \> 0) { std::string numberStr = std::to\_string(number); int count = 0;for (int i = 2; i \< numberStr.length(); i++) {if ((numberStr[i - 2] == '6' && numberStr[i - 1] == '6') && numberStr[i] == '6') {N--;break;}} number++;} std::cout \<\< number - 1;}

근데 이걸 디버거로 풀면 N이 10000일 때 기준으로 실행시간 4초 가량이 나온다.

주어진 시간제한은 2초인데, 왜 통과가 되는진 모르겠다.
