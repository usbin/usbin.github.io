 https://www.acmicpc.net/problem/10815


큰 범위의 많은 수 안에서 특정 수들을 찾아내야 하는 문제다.

수는 -1천만~+1천만까지의 범위를 갖고,

수의 개수는 50만개 이하이다.

​

이 문제를 푸는 방법은 여러가지가 있을 텐데,

  1. c++ 내장 find 함수를 사용하기

  2. 전체 범위를 담은 bool 배열로 풀기

  3. 이진탐색 구현하기

정도가 있을 것이다.

​

1. c++ 내장 find 함수를 사용하기

1번으로 풀었더니 시간초과가 났다.

N번의 순회를 도는 for문 안에 find 함수를 넣어야 하는데,

내장 find 함수는 O(n)의 시간복잡도를 갖는다.

따라서 이 방법은 O(n^2)의 시간복잡도를 갖는다.

이 방법은 패스.

​

2. 전체 범위를 담은 bool 배열로 풀기

이 방법은, -1천만부터 +1천만까지를 bool 배열에 담고, 있는 카드만 true로 바꿔주는 것이다.

그런 다음 입력되는 숫자를 인덱스로 array[input]이 true인지 false인지만 체크해서 출력하면 된다.

-1천만부터 +1천만까지 포함되는 범위이므로, 20,000,001칸의 배열이 필요하다.

​

입력되는 숫자엔 음수가 있는 반면 배열의 인덱스는 음수가 될 수 없으므로,

상수 k를 선언해서 1천만을 할당해준다.

이제 앞으로 배열의 인덱스는 array[k+input]으로 접근하면 된다.

소스코드는 다음과 같다.

#include <iostream> #include <algorithm> int k = 10000000; int main(){ std::cin.tie(NULL); std::ios::sync_with_stdio(false); bool cards[20000001]; std::fill_n(cards, 20000002, false); int N, M; std::cin>>N; for(int i=0; i<N; i++){ int input; std::cin>>input; cards[input+k] = true; } std::cin>>M; for(int i=0; i<M; i++){ int input; std::cin>>input; if(cards[input+k]) std::cout<<1<<" "; else std::cout<<0<<" "; } }



메모리를 20MB나 잡아먹지만 어쨌든 통과.

​

하지만 이건 좀 좋지 못한 방법이다.

숫자의 개수보다 범위가 훨씬 크기 때문에, 최악의 경우 N이 1개라면 고작 1개의 값을 저장하기 위해 2천만 1칸의 배열을 사용해야 한다.

따라서 가장 좋은 방법인 이진탐색을 쓰도록 하자.

​

3. 이진탐색을 사용하기

이진탐색이란 정렬된 배열의 중간 인덱스 값을 비교한 뒤,

더 작으면 아래쪽 절반에 대해 찾고,

더 크면 위쪽 절반에 대해 찾고,

같으면 리턴하는 방식의 탐색 방법이다.

​

코드로 보면 다음과 같다.

int binary_search(int* array, int first, int last, int x) { if (last < first) return -1; int mid = first + (last - first + 1) / 2; if (array[mid] > x) { return binary_search(array, first, mid - 1, x); } else if (array[mid] < x) { return binary_search(array, mid + 1, last, x); } else { //일치할 경우 return mid; } }

재귀호출보다 for문을 돌리는 게 더 효율적이지만 재귀가 식을 세우는 데 더 직관적이고 이걸로도 충분히 시간 복잡도가 성립되기 때문에 재귀함수로 만들었다.

​

전체 소스코드는 다음과 같다.

#include <iostream> #include <algorithm> int binary_search(int* array, int first, int last, int x) { if (last < first) return -1; int mid = first + (last - first + 1) / 2; if (array[mid] > x) { return binary_search(array, first, mid - 1, x); } else if (array[mid] < x) { return binary_search(array, mid + 1, last, x); } else { //일치할 경우 return mid; } } int main() { std::cin.tie(NULL); std::ios::sync_with_stdio(false); int N; std::cin >> N; int* cards = new int[N]; for (int i = 0; i < N; i++) { int input; std::cin >> input; cards[i] = input; } std::sort(cards, cards + N); int M; std::cin >> M; for (int i = 0; i < M; i++) { int input; std::cin >> input; int index = binary_search(cards, 0, N-1, input); if (index != -1) std::cout << 1 << " "; else std::cout << 0 << " "; } }

이진탐색의 시간복잡도는 O(log n)이므로, 이 방법으로 풀면 O(nlog n)의 시간복잡도를 갖게 되어 시간초과가 나지 않는다.



​



실행시간은 위와 비슷하지만 메모리가 1/5 가량으로 줄었다.
