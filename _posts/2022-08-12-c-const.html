 참조형과 포인터가 자주 헷갈려서 정리하려 한다.

일단 참조형은 실제로 const 포인터와 내부적으로도 같다고 한다.

const 포인터란, 포인터가 가리키는 값을 초기화할 때 설정한 후 변경할 수 없는 포인터를 말한다.

가령,

int a = 10; int* const p = &a; // a == *p 이고, &a == p 이다. int b = 20; p = &b; //컴파일 에러!

위의 코드를 보면 p를 처음 초기화할 때 a의 주소값을 할당한다.

이제 p를 역참조한 값은 a와 같고, a의 주소값은 p의 값과 같게 된다.

​

아무튼 본론으로 들어가서.



1. 참조형 변수 &

본래 변수와 완전히 똑같다고 봐도 무방.

int a = 10; int& ref = a; //참조형 변수는 앞에 &를 붙여 선언한다.

이제 a와 ref는 완전히 동일하다. ref에 값을 더하거나 빼도 a가 똑같이 변한다.

(*참고로, 여기서 &는 포인터를 초기화할 때 변수의 주소값을 가리키는 &와는 아예 다른 의미로 쓰인다.)

덧붙여, 참조형은 반드시 선언과 동시에 초기화가 이루어져야 한다.

​

이 참조형은 함수 파라미터로 값을 전달할 때 유용하게 쓰인다.

void func(int (&arr)[4]){ //arr에 접근 가능. } int main(){ int arr[] = {1, 2, 3, 4}; func(arr); }

위와 같은 방식으로 전달하면 장점이 두 가지가 있다.

1. 기본형 변수(int, float 같은)나 struct 타입일 경우, 변수를 파라미터로 전달하면 값의 복사가 일어나는데, 받을 때 &를 붙여 받으면 복사가 일어나지 않아 메모리와 속도에 이점이 있다.

참고) 참조형을 사용하지 않고 기본형 변수를 넘길 때

void func(int a){ //이때 main함수에서 넘겨준 파라미터 a가 func의 a로 복사됨. } int main(){ int a = 10; func(a); }

(int 하나 정도의 값은 사실 큰 차이가 없지만 가령 몇만 개 이상의 데이터를 담고 있는 Vector를 넘기거나 할 경우 차이가 굉장히 커진다.)

​

2. 배열의 경우, 파라미터로 전달하면 함수 안에서 포인터로 받아야 하는데, &를 사용하면 본래 타입 그대로 사용할 수 있다. 다만 사이즈를 함수 안에서 명시해줘야 한다. 따라서 배열의 크기가 동적일 경우엔 참조형을 사용할 수 없다.

참고) 참조형을 사용하지 않고 배열을 넘길 때

void func(int *arr){ //함수에 넘긴 후부터 arr은 포인터변수로 다뤄야 한다. } int main(){ int arr[] = {1, 2, 3, 4}; func(arr); }

2. 포인터 변수 *

int a = 10; int b = 20; int *p = &a; //선언과 동시에 초기화 p = &b; //재할당

참조형 변수와 가장 큰 차이점은 일단 처음에 초기화하고나서도 다른 값을 가리키도록 바꿀 수 있다는 것이다.

그리고 참조형 변수는 별다른 기호를 붙이지 않아도 참조중인 변수와 똑같이 다뤄지는 반면,

포인터 변수는 역참조 연산자 *를 붙여줘야 참조중인 변수와 같은 역할을 한다.

*p = 21; //b가 21이 됨.

​

또한, 포인터 변수는 const 한정자를 붙이면 (1) 참조 대상을 변경할 수 없게 만들거나 (2) 참조 대상의 값을 변경할 수 없게 만들 수도 있다.

(1) int* const p1

int* const p1 = &a; p1 = &b; //에러!

이 포인터는 앞으로 다른 변수를 가리킬 수 없다. 참조형 변수와 같은 역할을 한다.

​

(2) const int* p2

const int* p2 = &a; *p2 = 11; //에러! p2 = *b; //이건 가능.

이 포인터는 참조 변수의 값을 변경할 수 없다.

참조중인 대상을 바꾸는 건 가능하다.

​

​

더 나아가, 포인터 변수는 이차원 포인터, 삼차원 포인터 등 다중 차원 포인터를 만들 수도 있다.

기본적으로 int*가 int형 변수를 가리킨다면, int**는 int형 변수를 가리키는 포인터를 가리킨다.

int** -> int* -> int

​

또한, 포인터에 덧셈을 하면 본인이 가리키는 값의 타입의 사이즈만큼을 자동으로 계산해, 다음 값을 가리키게 된다.

int arr[] = {1, 2, 3, 4}; int *p = arr;

위의 경우에, 배열의 이름인 arr은 배열의 시작 주소를 가리킨다. p 또한 arr의 시작 주소를 가리키게 된다.

*p는 arr의 시작 주소에 담긴 int값을 읽어오므로 1이 된다.

p+1은 자동으로 p의 값에 sizeof(int) * 1만큼을 더한 것이 되어, 숫자 2가 저장된 주소를 가리킨다.

*(p+1)로 역참조하면 2가 된다.

즉, *(p + i)로 i번 인덱스의 값에 접근할 수 있다.

​
