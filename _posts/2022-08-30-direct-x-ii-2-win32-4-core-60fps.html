### 1. 싱글톤 만들기(Core 클래스)

유니티에서 싱글톤을 만들 땐 static타입 인스턴스 변수를 클래스의 멤버변수로 선언하는 식으로 했는데
이 강의에선 로컬 static 변수로 선언해서 반환해주더라.

정적 멤버변수로 구현하는 게 좀더 직관적으로 느껴지지만
접근을 반드시 GetInstance를 통해서만 하도록 하기 위해서 똑같이 정적 로컬변수로 구현해주었다.

```c++
//Core.h
#pragma once

class Core {
public:
	static Core* GetInstance();
private:
	Core();
	~Core();
};
```

```c++
//Core.cpp
#include "Core.h"

Core* Core::GetInstance()
{
	static Core instance;
	return &instance;

}

Core::Core() {}
Core::~Core() {}

```

이렇게 하면 싱글톤으로 인스턴스를 다루는 클래스가 생성된다.
private으로 생성자와 소멸자가 선언되었으므로 외부에서 새로 생성하는 건 불가능하고, GetInstance()를 통해서만 인스턴스를 얻을 수 있는데
이 인스턴스는 정적 로컬변수이므로 프로세스가 시작될 때 생성된 후 프로세스가 종료될 때까지 매번 같은 값을 반환한다.

싱글톤은 자주 사용할 것이기 때문에 강의와 똑같이 '미리 컴파일된 헤더'를 사용해서 매크로로 빼두었다.

덧붙여 정적 변수는 프로그램이 돌아가는 도중에 자원을 반납할 수 없어서 최대한 쓰지 않는 게 좋지만,
싱글톤 클래스는 프로세스의 시작부터 끝까지 절대 반환하지 않을 자원에 할당하는 용도로 쓰이기 때문에 프로그램의 중심이 될 Core 클래스의 인스턴스를 정적 변수로 만드는 듯하다.


|미리 컴파일된 헤더 : 모든 소스파일에서 공통적으로 사용할 헤더를 하나의 헤더파일에 include해둔 뒤, 해당 헤더파일을 "미리 컴파일 된 헤더"로 설정하면 컴파일 속도가 올라감.||
|---|---|


먼저 폴더 구조부터 정리하고




사진처럼 Header 폴더 안에 macro.h와 pch.h라는 헤더파일을 새로 생성했다.
macro.h 안에는 싱글톤 매크로를 넣고

```c++
#pragma once

#define SINGLETON(type) public:\
							static type* GetInstance()\
							{\
								static type instance;\
								return &instance;\
							}
```

pch.h 안에는 macro.h를 include하는 코드를 넣었다.

```c++
#pragma once

#include "macro.h"
```

그리고 프로젝트-속성-C/C++-미리 컴파일된 헤더 를 차례로 들어가 미리 컴파일된 헤더를 "만들기"로 설정한 후, 이름을 pch.h로 설정해준다.


이제 모든 소스파일에서 pch.h를 include해주면 SINGLETON 매크로를 사용할 수 있게 된다.

적용해보자.

일단 기본 소스파일(본 프로그램의 경우 Win32Program.cpp) 최상단에 include pch.h를 해주고,

Core.h파일을 SINGLETON 매크로를 사용하도록 수정해준다.

```c++
//Core.h
#pragma once
class Core {
	SINGLETON(Core);
private:
	Core();
	~Core();
};

```

Core.cpp 파일은 미리 컴파일된 헤더와 Core.h를 포함한다.
그리고 생성자와 소멸자를 private로 선언해준다.(밖에서 생성하지 못하게 하려는 용도이므로 빈 함수로 두면 된다)

```c++
//Core.cpp
#include "pch.h"
#include "Core.h"

Core::Core() {}
Core::~Core() {}

```

대충 이런 느낌이다.

### 2. C스타일 함수와 C++스타일 함수

C스타일 함수는

```c++
Item item;
InitItem(&Item, itemCode, itemCount);
```
이런 식이고

C++스타일 함수는

```c++
Item item;
item.Init(itemCode, itemCount);
//혹은
item = InitItem(itemCode, itemCount);
```
이런 식이라고 한다.

그러니까 C 스타일은 함수의 첫번째 인자로 '다룰 객체'를 넣어주는데, C++은 객체의 멤버함수를 그냥 실행시켜버리거나 리턴값으로 받는다는 차이가 있다.
리턴값으로 받을 경우, 이 변수가 작을 때에만 유용하다. 큰 데이터를 리턴값으로 받으면 오버헤드가 크므로 이런 경우엔 그냥 C타입처럼 인자로 받는 게 좋다.



### 3. AdjustWindowRect, SetWindowRect

윈도우의 크기를 설정하는 함수이다.
SetWindowRect 함수는 창의 테두리나 메뉴탭까지 포함한 전체 윈도우 크기를 설정하는데,
프로그램 내에서의 좌표계는 테두리와 메뉴탭을 제외한 작업영역에서만 작용한다.
따라서 작업영역을 1000, 1000 크기로 만들고 싶을 때 SetWindowRect에 1000, 1000을 넣어주면 실제 작업영역은 테두리와 메뉴탭을 뺀 크기가 되어 의도한 것보다 작아지게 된다.

원하는 크기의 작업영역을 만들려면 처음에 Rect 변수를 선언해서 left, top, right, bottom 값을 원하는 작업영역 크기로 넣어준 뒤
AdjustWindowRect()함수에 옵션과 함께 Rect 변수를 넣으면 창 테두리나 메뉴탭에 필요한 사이즈를 더해서 더 큰 값으로 만들어준다.
이 결과값을 setWindowRect에 넣어서 윈도우를 만들면 원하는 크기가 나온다.

```c++
RECT rect{ 0, 0, 1000, 1000 };
AdjustWindowRect(&rect, WS_OVERLAPPEDWINDOW, TRUE)
SetWindowPos(hWnd, nullptr, 100, 100, rect.right - rect.left, rect.bottom - rect.top, 0)
```
AdjustWindow의 (Rect 변수, 윈도우 타입, 메뉴탭을 만들 것인지 여부)를 파라미터로 받고
SetWindowPos는 (윈도우 핸들, After윈도우 핸들, 시작 위치 x, 시작 위치 y, width, height, 옵션) 을 파라미터로 받는다.
After윈도우 핸들과 옵션은 널값을 넣고 나머지만 채워주면 된다.


### 4. Core 클래스

이걸 Core 클래스의 Init함수에 집어넣었다.
앞으로 프로그램이 시작되면 먼저 Init을 하면서 윈도우 핸들을 받고 윈도우 크기를 지정해줄 것이다.

```c++
//Core.h
class Core {
	SINGLETON(Core);
	int Init(HWND hwnd, int width, int height);
private:
	HWND h_wnd_;
	Core();
	~Core();
};
```

Core.h에 멤버변수로 h_wnd_를 추가해주고

```c++
//Core.cpp
int Core::Init(HWND hWnd, int width, int height) {
	h_wnd_ = hWnd;
	RECT rect{ 0, 0, width, height };
	if (!AdjustWindowRect(&rect, WS_OVERLAPPEDWINDOW, TRUE)) {
		return E_FAIL;
	}
	if (!SetWindowPos(hWnd, nullptr, 100, 100, rect.right - rect.left, rect.bottom - rect.top, 0)) {
		return E_FAIL;
	}
	return S_OK;
}
```
인자로 hWnd를 받아 초기화해준다.


메인 소스파일인 Win32Program.cpp에서도 전역변수로 g_hWnd를 선언하고 InitInstance 함수 안에서 g_hWnd를 사용하는 코드로 바꾼다.

```c++
BOOL InitInstance(HINSTANCE hInstance, int nCmdShow)
{
   hInst = hInstance; // 인스턴스 핸들을 전역 변수에 저장합니다.

   g_hWnd = CreateWindowW(szWindowClass, szTitle, WS_OVERLAPPEDWINDOW,
      CW_USEDEFAULT, 0, CW_USEDEFAULT, 0, nullptr, nullptr, hInstance, nullptr);

   if (!g_hWnd)
   {
      return FALSE;
   }

   ShowWindow(g_hWnd, nCmdShow);
   UpdateWindow(g_hWnd);

   

   return TRUE;
}
```

### 5. DC 얻기: BeginPaint와 GetDC의 차이점

BeginPaint도 DC를 반환하고 GetDC도 DC를 반환한다.
둘의 차이점은 BeginPaint로 DC를 받았을 경우 사용이 끝나면 EndPaint로 반환해야 하며, 메시지 처리기에서만 사용이 가능하다.

- BeginPaint: 무효화 영역의 시작을 의미
- EndPaint: 무효화 영역의 끝을 의미. WM_PAINT 메시지를 삭제시킴.

이와 달리 GetDC는 메시지 처리기 밖에서도 사용이 가능하며 사용자가 BeginPaint같은 함수로 시작시키는 게 아니기 때문에 프로그램이 종료될 때 한 번 ReleaseDC로 반환해주면 된다.

### 7. GetAsyncKeyState()

메시지 처리기에서는 wParam값에 눌린 키 값이 들어왔는데, Core.Process 함수는 메시지처리기 밖에 있기 때문에 wParam을 얻을 수가 없다.
또한 메시지 처리기는 윈도우에 포커싱되어 있을 때에만 메시지를 받기 때문에, 다른 프로그램이 포커싱 된 상태에서는 입력을 받을 수 없다.

GetAsyncKeyState()는 현재 포커싱된 윈도우에 관계없이 사용자의 입력을 받을 수 있는 함수이다.

Process() 함수에 넣고 입력을 받아보자.

```c++
bool Core::Process()
{
	SHORT is_up = GetAsyncKeyState(VK_UP)&0x8000;
	if (is_up) MessageBox(h_wnd_, _T("위 키가 눌림!"), _T("알림"), MB_OK);

	Update();
	Render();

	return TRUE;
}
```

키보드 위 키를 누르면 메시지박스가 뜨는 걸 확인할 수 있다.



VK_UP은 위쪽 화살표를 의미하는데, 키보드의 모든 값은 https://docs.microsoft.com/ko-kr/windows/win32/inputdev/virtual-key-codes 에서 볼 수 있다.
또한 리턴값은 16bit로 반환되는데, 여기서 파싱할 수 있는 건 두 가지다.

최상위 비트가 1이면(0x8000) -> 현재 눌려있다.
최하위 비트가 1이면(0x0001) -> 직전에도 눌렸다.

라고 가이드에선 말하는데, 테스트해보니 그냥 0x8000은 계~속 누르고 있는 중일 때 1이 되고, 0x0001은 처음 눌렀을 때에만 1이 되는 것 같다.

그러니까 위 키를 꾹 누르고 있으면 GetKeyState(VK_UP)이 최초 1번은 0x8001이 된 후 두번째부턴 계속 0x8000이 되고,
위 키를 한 번씩 탁 탁 누르면 0x8001만 입력되는 것이다.

쉽게 말해 0x8000은 "키가 꾹 눌려있는 상태", 0x8001은 "키가 눌림" 으로 보면 될 듯하다.



### 8. 시간 동기화

아까 작성한 Process 함수는 메시지 처리기에서 PeekMessage가 FALSE를 반환하면(=윈도우 메시지가 없으면) 계속해서 실행되게 되어있다.
윈도우 메시지가 발생하는 시간에 비해 발생하지 않는 빈 시간이 훨씬 많기 때문에(사실상 프로그램이 돌아가는 동안 99%의 시간이 여기에 속함)
실제로 Process 함수는 프로그램을 시작하면 쉴새없이 많이 호출되고 있다.

> Process 함수의 실행 빈도는 곧 화면이 새로 그려지는 빈도라는 뜻인데, 보통 게임은 60fps 정도로 화면을 고친다. 1초에 60번 고친다는 뜻이다.
> 낮으면 30fps, 높으면 144fps로 그리기도 하는데 60fps를 넘어가면 사실 육안으로 구분하기 힘들다고 한다.


아무 처리도 하지 않았을 때 1초에 몇 번 실행되는지 확인한 후, 이 실행 횟수를 n으로 잡았을 때,
0부터 함수를 실행할 때마다 카운트를 세어서 n에 도달하면 1초가 지났다는 뜻이다.

QueryPerformanceFrequence() 함수는 사용하면 1초에 몇 틱이 실행되는지 알려준다.
가령 어떤 코드를 실행하는 데 100틱이 걸렸고, 초당 틱이 10틱이라면, 실제 실행 시간은 100/10 = 10초가 걸린 것이다.

QueryPerformanceCounter() 함수는 현재 틱 카운트를 알려준다. 시작 지점과 끝 지점의 차를 구한 뒤 주파수로 나눠주면 실제 걸린 시간을 초 단위로 구할 수 있다.



t를 인자로 받아서 확인해보니 10,000,000이라는 값이 잡힌다.
1초에 천만 번의 틱이 실행된다는 뜻이다. 다른 말로, 천만 번의 틱이 실행됐다면 시간이 1초가 지났다는 걸 의미한다.

이제 함수가 실행될 때마다 틱 카운트를 세어주면서 틱 카운트가 1/60초가 될 때마다 Update()와 Render() 함수가 실행되게 할 것이다.


전역변수로 frequency와 이전 틱수를 저장할 count 변수를 선언해주고,
dt와 누적 dt, 그리고 fps도 선언해준다.

```c++
//Core.h
	LARGE_INTEGER query_performance_frequency_; //초당 틱수
	LARGE_INTEGER prev_tick_count_;				//이전 틱 카운트
	double dt_;									//dt=1프레임에 걸리는 시간
	double acc_dt_;								//누적 dt
	int fps_;									//Frame Per Second : 초당 프레임 수
												//- 1/dt로 구하지 않는 이유: 정확도
```

초기화는 Init() 함수에서 해준다.

```c++
//Core.cpp
	QueryPerformanceFrequency(&query_performance_frequency_); //1초당 틱수
	QueryPerformanceCounter(&prev_tick_count_); //현재 틱수
	dt_ = acc_dt_ = 0.f;
	fps_ = 0;
    //... 생략
}
```

이제 Update() 함수에서 누적 dt를 더해주면서 누적 dt가 1000ms(=1초)가 될 때마다 프로그램 타이틀에 문자열을 출력하도록 한다.

```c++
bool Core::Update()
{
	LARGE_INTEGER current_count;
	QueryPerformanceCounter(&current_count);
	dt_ = static_cast<double>(current_count.QuadPart - prev_tick_count_.QuadPart)*1000 / query_performance_frequency_.QuadPart ;
	acc_dt_ += dt_;
	fps_++;

	if (acc_dt_ >= 1000.f) {
		TCHAR buffer[100];
		_stprintf_s(buffer, _T("현재 FPS: %d, DT: %f"), fps_, dt_);
		SetWindowTextW(h_wnd_, buffer);
		acc_dt_ = 0.f;
		fps_ = 0;
	}
	prev_tick_count_.QuadPart = current_count.QuadPart;
	
	return TRUE;
}
```
실행하면 1초마다 타이틀이 바뀌는 걸 확인할 수 있다.



이제 이건 Timer라는 클래스로 따로 정의해서 정리하고 빼둘 것이다.
앞으로 어떤 키 입력에 따른 움직임 등 FPS에 영향을 받는 동작을 할 때 dt를 곱해주면 모든 하드웨어 환경에서 동일하게 수행됨을 보장할 수 있다.
(느린 컴퓨터는 dt가 크고 빠른 컴퓨터는 dt가 작을 것이므로)
