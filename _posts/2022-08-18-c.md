    # 스마트 포인터란?
    
    스마트 포인터란 Java나 C#의 가비지 컬렉션을 대체하는 것으로, 사용하지 않게 된 메모리를 알아서 해제해주는 포인터를 말한다.
    use_count()로 현재 몇 개의 포인터가 같은 메모리를 참조중인지도 알 수 있다.
    
    스마트포인터를 선언하는 코드는 세 종류가 있다.

    ## 1. unique_ptr
    
    unique_ptr은 이름대로 딱 하나만 있는 포인터다. 어떤 메모리공간을 가리키는 포인터는 유일하게 하나만 존재한다.
    
    std::make_unique<클래스명>(생성자 파라미터) 로 생성할 수 있다.
    
    따라서 다른 포인터로 옮길 땐 std::move(ptr) 함수를 사용해야 하며, 옮긴 후부터 이전 포인터에 접근하려 하면 런타임 에러가 난다.
    대입, 복사 모두 사용할 수 없고 함수 파라미터로도 그냥 넘기지 못한다.
    
    
    ## 2. shared_ptr
    
    shared_ptr은 여러개가 존재할 수 있다. std::make_shared<클래스명>(생성자 파라미터)로 생성할 수 있다.
    대입, 복사, 함수 파라미터로 모두 사용 가능하며, 마지막 포인터가 사라질 때까지 메모리가 해제되지 않는다.
    
    
    
    ## 3. weak_ptr
    
    weak_ptr은 make_어쩌고로 생성할 수 없고 이미 존재하는 다른 포인터에서 복사할 수만 있다.
    참조중인 포인터로 카운트되지 않는다.
    
    
    ```c++
    //사용 예제
    #include <iostream>
    #include <memory>
    
    class SomeClass {
    	int a, b;
    public:
    	SomeClass(int a, int b) {
    		this->a = a;
    		this->b = b;
    		std::cout << "생성자 호출됨!\n";
    	}
    	~SomeClass() {
    		std::cout << "소멸자 호출됨!\n";
    	}
    };
    void use_unique_ptr() {
    	std::cout << "<---use_unique_ptr 시작!\n";
    	std::unique_ptr<SomeClass> ptr = std::make_unique<SomeClass>(1, 2);
    	std::unique_ptr<SomeClass> ptr2 = std::move(ptr); //옮길 땐 복사, 대입 불가. move 함수로만 가능.
    	std::cout << "<---use_unique_ptr 종료!\n";
    }
    
    void use_shared_ptr(std::shared_ptr<SomeClass> ptr) {
    	std::cout << "<---use_shared_ptr 시작!\n";
    	std::cout << "현재 참조 개수:" << ptr.use_count() << "\n";
    	std::cout << "<---use_shared_ptr 종료!\n";
    }
    int main() {
    	std::cout << "1. unique_ptr 사용\n";
    	use_unique_ptr();
    	std::cout << "\n";
    
    	std::cout << "2. shared_ptr 사용\n";
    	std::shared_ptr<SomeClass> s_ptr = std::make_shared<SomeClass>(3, 4);
    	std::cout << "현재 참조 개수:" << s_ptr.use_count()<<"\n";
    	std::weak_ptr<SomeClass> w_ptr = s_ptr;
    	use_shared_ptr(s_ptr);
    	std::cout << "현재 참조 개수:" << s_ptr.use_count()<<"\n";
    	std::cout << "<---main 함수 종료!\n";
    
    }
    
    ```

    **unique_ptr** 의 경우, 함수 안에서 포인터를 선언하고 메모리를 가리켰기 때문에 함수가 종료되면 포인터 변수가 사라지며 포인터가 가리키고 있던 SomeClass도 자동으로 소멸한다.
    
    **shared_ptr** 의 경우, 처음엔 use_count()가 1이었다가 함수 안에서 파라미터로 받으며 복사가 한 번 이뤄졌기 때문에 함수 안에 진입하면 use_count()가 2가 되고, 함수가 끝나면 다시 1로 돌아온다.
    그리고 함수가 끝나도 포인터는 사라지지 않고 main 함수가 종료될 때가 돼서야 소멸자가 호출되는 것을 확인할 수 있다.
    
    덧붙여 참조 개수를 보면 알 수 있듯이, use_count()는 **weak_ptr** 를 무시하고 카운트한다.
