### 1. WM_PAINT

이전 포스팅에서 Win32 프로그램은 초기화와 메시지 루프로 이루어진다고 했었다.
여기서 메시지 루프의 구조를 보면

```c++
LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam)
{
    switch (message)
    {
    case WM_COMMAND:
        {
            int wmId = LOWORD(wParam);
            // 메뉴 선택을 구문 분석합니다:
            switch (wmId)
            {
            case IDM_ABOUT:
                DialogBox(hInst, MAKEINTRESOURCE(IDD_ABOUTBOX), hWnd, About);
                break;
            case IDM_EXIT:
                DestroyWindow(hWnd);
                break;
            default:
                return DefWindowProc(hWnd, message, wParam, lParam);
            }
        }
        break;
    case WM_PAINT:
        {
            PAINTSTRUCT ps;
            HDC hdc = BeginPaint(hWnd, &ps);
            // TODO: 여기에 hdc를 사용하는 그리기 코드를 추가합니다...
            EndPaint(hWnd, &ps);
        }
        break;
    case WM_DESTROY:
        PostQuitMessage(0);
        break;
    default:
        return DefWindowProc(hWnd, message, wParam, lParam);
    }
    return 0;
}
```
메시지가 WM_PAINT일 경우, 뭔가를 그리는 코드를 추가할 수 있다.
이 코드는 최초에 1번, 그리고 윈도우가 최소화 되었다가 다시 돌아올 때마다 실행된다.
Rectangle(핸들, x, y, width, height) 함수 등으로 윈도우에 그림을 그릴 수 있다.

hdc는 Device Context로, 그리기에 필요한 핸들이다.
펜: 기본값 검정
브러쉬(채우기) : 기본값 하양


### 2. 그리기 과정

1. BeginPaint로 시작한다.
2. 그린다.
3. EndPaint로 끝낸다.

코드를 보면 그리기가 위와 같은 형태로 이루어지는 걸 알 수 있다.
BeginPaint는 뭘까?
https://docs.microsoft.com/en-us/windows/win32/gdi/painting-and-drawing-functions
링크를 참조하면 MS에서 제공하는 그리기 함수들을 볼 수 있는데, 가장 위에 BeginPaint가 있다.


```c++
HDC BeginPaint(
  [in]  HWND          hWnd,
  [out] LPPAINTSTRUCT lpPaint
);
```
함수는 이런 식으로 이루어져 있다. in은 받는 인자, out은 뭔가 값을 넣어서 채워준다는 뜻이다.
일전에 포스팅에서 LP는 Long Pointer를 뜻한다고 그랬으니 두번째 인자는 PAINTSTRUCT를 가리키는 포인터다.
구조를 보니
```c++
typedef struct tagPAINTSTRUCT {
  HDC  hdc;
  BOOL fErase;
  RECT rcPaint;
  BOOL fRestore;
  BOOL fIncUpdate;
  BYTE rgbReserved[32];
} PAINTSTRUCT, *PPAINTSTRUCT, *NPPAINTSTRUCT, *LPPAINTSTRUCT;

```
HDC(디바이스 핸들)과 여러 변수들을 가지고 있다.
그러니까, BeginPaint에 첫번째 인자로 윈도우 핸들을 넣으면 리턴 값으로 디바이스 핸들인 hdc, 두번째 인자로 페인팅에 필요한 구조체 ps를 받는다는 거다.
그리기를 종료할 때에도 hdc와 ps를 넣어 종료한다.


### 3. 네모 그려보기


https://docs.microsoft.com/ko-kr/windows/win32/gdiplus/
위 링크를 참조하면 쓸 수 있는 함수가 굉장히 많은데 저걸 다 익혀야 할지 잘 모르겠지만 일단 가장 간단한 Rectangle을 사용해봤다.
```c++
case WM_PAINT:
{
	PAINTSTRUCT ps;
    HDC hdc = BeginPaint(hWnd, &ps);
    
    Rectangle(hdc, 10, 10, 100, 100);
            
    EndPaint(hWnd, &ps);
}
        break;
```


프로그램을 실행하면 네모가 그려진 창이 하나 뜬다.
좌표는 좌상단이 (0, 0)인 듯하다.


CreatePen은 첫번째 인자로 펜의 모양, 두번째 인자로 펜의 너비, 세번째 인자로 색깔을 받는다.
***펜의 모양***은


https://docs.microsoft.com/en-us/windows/win32/api/wingdi/nf-wingdi-createpen
여기에서 확인할 수 있고, 종류는 사진과 같다.

***펜의 너비***는 펜의 모양이 PS_SOLID(직선)일 때만 쓰이는 것으로, 펜의 굵기를 지정한다.
만일 펜의 모양이 PS_SOLID가 아닌데 너비를 2 이상의 값을 지정하면 솔리드로 자동으로 바뀌어버린다.
0을 넣으면 자동으로 1 pixel이 된다고 한다.

***색깔***은 RGB(...)함수로 넣어주면 된다. 알파값은 쓰이지 않는 것 같다.

색깔과 선의 종류를 한 번 바꿔보자.

왼쪽은 
```c++
PAINTSTRUCT ps;
HDC hdc = BeginPaint(hWnd, &ps);

HPEN pen, oldPen;
pen = CreatePen(PS_DOT, 1, RGB(255, 0, 0)); //점선, 빨간색
oldPen = (HPEN)SelectObject(hdc, pen);
Rectangle(hdc, 10, 10, 100, 100);

EndPaint(hWnd, &ps);
```

오른쪽은
```c++
PAINTSTRUCT ps;
HDC hdc = BeginPaint(hWnd, &ps);

HPEN pen, oldPen;
pen = CreatePen(PS_DASH, 1, RGB(255, 0, 0)); //대쉬선, 빨간색
oldPen = (HPEN)SelectObject(hdc, pen);
Rectangle(hdc, 10, 10, 100, 100);
            
EndPaint(hWnd, &ps);
```
의 소스코드로 출력한 결과이다.
CreatePen에 넣은 인자에 따라 그려진 결과가 바뀌는 걸 알 수 있다.


### 4. 비트맵 이미지 그려보기

단순한 도형을 그리는 것 말고 이미지를 그릴 수도 있다.

비트맵 이미지를 불러온 다음, 메모리DC에 그린 뒤, 화면DC에 고속복사해서 보여주는 방식이다.
메모리DC는 우리 눈에 보이지 않는 가상 화면이라고 생각하면 될 것 같다.
바로 화면DC에 보여주지 않는 이유는 비트맵 이미지를 불러오는 데 시간이 꽤 걸리기 때문에 불러와서 출력하는 동안 딜레이가 걸리기 때문이다.
먼저 메모리DC에 불러오게 하면 그동안 다른 작업을 할 수 있고, 그 뒤 메모리에서 바로 화면으로 복사하는 건 아주 빠르게 수행 가능하다.

- 아무튼 일단 아무거나 비트맵을 하나 디렉토리에 넣어보자.


vs 코드 화면을 캡쳐한 사진을 bmp 확장자로 저장했다.
이제 이걸 프로그램에 불러와보자.

새 항목을 만든 뒤 리소스 파일(.rc)을 선택하고 이름을 Bitmap.rc로 지정한 뒤 생성한다.

위처럼 추가되었다면 리소스 뷰에서 우클릭해 리소스 추가를 해준다.


  
솔루션 탐색기에서 리소스에 추가해준 다음, 코드를 작성한다.
```c++
PAINTSTRUCT ps;
HDC hdc = BeginPaint(hWnd, &ps);

HDC memDC;
HBITMAP bitmap, oldBitmap;
BITMAP bit;
memDC = CreateCompatibleDC(hdc);//메모리DC 생성
bitmap = LoadBitmap(hInst, MAKEINTRESOURCE(IDB_BITMAP1)); //비트맵 로드
GetObject(bitmap, sizeof(BITMAP), &bit); //비트맵 정보 가져옴->bit

oldBitmap = (HBITMAP)SelectObject(memDC, bitmap); //메모리DC의 비트맵에 넣고 기존 비트맵(빈 비트맵)을 받아옴.
BitBlt(hdc, 10, 10, bit.bmWidth, bit.bmHeight, memDC, 0, 0, SRCCOPY); //메모리DC -> DC로 고속복사
SelectObject(memDC, oldBitmap); //메모리DC의 비트맵 복구
DeleteObject(bitmap); //불러온 비트맵 삭제
DeleteDC(memDC); //메모리DC 삭제

EndPaint(hWnd, &ps);
```
여기서 SelectObject 함수는 DC 안의 GDI 오브젝트를 교체하고 기존 오브젝트를 뱉어내는 함수인데, 교체 가능한 GDI 오브젝트로는 Bitmap, Pen, Brush, Font, Region이 있다. 그런데 Bitmap의 경우, 메모리DC일 경우에만 교체가 가능하고 화면 DC는 불가능하다고 한다.

좀전의 비트맵 그림이 프로그램에 제대로 그려지는 걸 확인할 수 있다.
