https://www.acmicpc.net/problem/9251


어떻게 매번 색다르게 어려운 문제가 나올까.
보기엔 참 단순해보이는데 코드로 짜려면 정신이 대략 멍해진다.
동적계획법이란 걸 아는데도.

접근하기

이 문제는 문자열 두 개를 비교하면서 최장 공통 부분 수열을 찾는 문제이다.
LCS라고도 한다.
최장 공통 부분 수열이란, 말로 정의하기가 참 애매한데, 문자열 두 개를 비교하면서 같은 순서로 문자가 나오는 부분을 찾는 것이다. 사이사이에 어떤 알파벳이 끼여있든 상관없이, 나오는 순서만 같으면 공통 부분 수열이 된다.

A와 A의 공통 부분수열은 "A"이고, 길이는 1이다.
AB와 ABC의 공통 부분수열은 "AB"이고, 길이는 2이다.
ABCD와 BEC의 공통 부분수열은 "B", "C", "BC"이고, 최장 부분수열인 "BC"의 길이는 2이다.

이런식이다. 두 문자열의 공통부분수열은 여러 개 존재할 수 있고, 우리는 거기서 최장 공통부분수열을 찾아야 한다.

이 문제는 점화식을 세우면 아주 단순해진다.
그냥 머리를 비우고 f(n)을 f(n-1)로 어떻게 만들까를 생각해보면 된다.

여기 공통부분수열의 길이를 담은 배열 DP[1001][1001]이 있다. (입력 문자열이 최대 1000글자까지이므로, 인덱스 0을 글자가 없을 때로 치고 인덱스 1부터 저장)
DP[i][j]는 str의 i번째 인덱스까지의 부분문자열과 str2의 j번째 문자열까지의 부분문자열 길이를 저장하는 배열이다.

ABCD와 BEC로 예를 들어보자.
DP[0][0]은 빈 문자열과 빈 문자열의 최장공통부분문자열의 개수인데, 0으로 설정해준다.
DP[0][1]부터 DP[0][1001]까지는 다 0이다.
DP[1][0]도 0이다.
DP[1][1]은 A와 B의 LCS이므로 0이다.
DP[1][2]는 A와 BE의 LCS이므로 0이다.
DP[1][3]은 A와 BEC의 LCS이므로 0이다.
DP[2][0]은 0이다.
DP[2][1]은 AB와 B의 LCS이므로 1이다.
DP[2][2]는 AB와 BE의 LCS이므로 1이다.
이런 식으로 채워나가는 것이다.
이걸 식으로 어떻게 세울까?

먼저, 어떤 문자열 둘의 LCS는 다음과 같이 나눠볼 수 있다.
(1) 마지막 두 문자가 같다면, 마지막 문자를 뺀 두 문자열의 LCS +1과 같고
(2) 마지막 두 문자가 다르다면, 마지막 문자를 뺀 두 문자열에서 문자열1의 마지막 문자를 선택했을 때와 문자열2의 마지막 문자를 선택했을 때 중 더 긴 공통부분수열이 나오는 쪽을 선택한 LCS와 같은 것이다.

즉,
(1) DP[i][j] = DP[i-1][j-1] + 1 (str1[i] == str2[j]일 때)
(2) DP[i][j] = max(DP[i-1][j], DP[i][j-1]) (str1[i] != str2[j] 일 때)
위와 같은 식을 세울 수 있다.

DP[i][j]를 채우기 위해선 DP[i-1][j-1], DP[i][j-1], DP[i-1][j]가 채워져있어야 한다.
즉, 왼쪽위부터 채워나가면 된다.

#include <iostream>
#include <string>
#include <cmath>
std::string str1;
std::string str2;
int LCS[1001][1001];
 
void Solve(){
    //LCS[i][j]
    //만약 문자열1의 i번째 값과 문자열2의 j번째 값이 같을 경우
    //기존의 최장 공통부분수열의 길이가 1 늘어난 것과 같음.
    //= LCS[i-1][j-1] + 1
    //값이 다를 경우
    //기존의 최장 공통부분수열에 문자열1의 i번째 문자를 포함한 것과
    //                        문자열2의 j번째 문자를 포함한 것 중 최장을 선택.
    //
    //= max(LCS[i][j-1],LCS[i-1],[j])
    
    for(int i=1; i<str1.size(); i++){
        for(int j=1; j<str2.size(); j++){
            if(str1[i] == str2[j]){
                LCS[i][j] = LCS[i-1][j-1]+1;
            }
            else{
                LCS[i][j] = std::max(LCS[i][j-1], LCS[i-1][j]);
            }
        }
    }
    std::cout<<LCS[str1.size()-1][str2.size()-1];
    
}
void Input(){
    char cstr1[1002], cstr2[1002];
    std::cin>>cstr1>>cstr2;
    //str의 1번 인덱스부터 문자를 채우기 위해 맨 앞에 의미없는 값을 하나 넣어줌.
    str1.push_back('\0');
    str2.push_back('\0');
    str1.append(cstr1);
    str2.append(cstr2);
    
}
int main(){
    Input();
    Solve();
}

참고로 배열도 LCS[1001][1001]이 아니라 LCS[1001]로 바꿔줘야 한다.
여기서 주의할 점이, '왼쪽 대각선 값'을 이전 순회에서 미리 저장해둬야 한다.
1차원 배열을 사용하면 왼쪽에서부터 값을 덮어씌우게 되는데, 덮어씌우기 전의 값이 [i-1]이다.
즉, [i-1][j-1]과 [i][j-1]은 다른데, 덮어씌우기 전에는 [i-1][j-1]이고 덮어씌운 후에는 [i][j-1]이 된다.
str1[i] == str2[j] 일 경우, [i-1][j-1]의 값을 참조해야 하는데 바로 직전 순회에서 LCS[j-1]은 이미 i일 때로 덮어씌워졌다. 따라서 이전 순회에서 값을 덮어쓰기 전에 저장해두고, 이 값을 한 순회씩 지연시켜 갱신하면서 이 값을 대각선 위쪽 값으로 써야 한다.

    //기존의 최장 공통부분수열에 문자열1의 i번째 문자를 포함한 것과
    //                        문자열2의 j번째 문자를 포함한 것 중 최장을 선택.
    //
    //= max(LCS[i][j-1],LCS[i-1],[j])
    
    //왼쪽 위, 위, 왼쪽 값을 쓰기 때문에 왼쪽 위부터 채워야 하는데
    //왼쪽을 채우면서 왼쪽 위가 덮어씌워지기 때문에 이 값을 전의 순회에서 저장해둬야함.
    
    for(int i=1; i<str1.size(); i++){
