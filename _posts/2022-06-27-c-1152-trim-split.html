이 문제 되게 쉽게 풀 수도 있던데(>> 오퍼레이터 사용하면 trim 안 해도 되고 split은 알아서 해준다)

지금은 c++ 문법 공부 목적이니까 이것저것 다 써보고 외워보기로 했음.

​

소스코드는 아래.

#include <iostream> #include <string> //std::string #include <sstream> //std::istringstream int main() { //공백으로 시작하거나 끝날 수 있는 문자열이 주어짐. //공백을 기준으로 분리, 단어의 개수를 센다. std::string sentence; std::getline(std::cin, sentence); sentence.erase(0, sentence.find_first_not_of(' ')); sentence.erase(sentence.find_last_not_of(' ')+1); std::istringstream iss(sentence); std::string word; int count = 0; while (std::getline(iss, word, ' ')) { count++; } std::cout << count; }

​

먼저

(1) 문자열 trim

sentence.erase(0, sentence.find_first_not_of(' ')); sentence.erase(sentence.find_last_not_of(' ')+1);

이건 sentence의 앞부분 공백과 뒷부분 공백을 자르는 함수이다.

이번 문제에선 앞뒤 공백이 각각 2개 이상 들어가지 않는다고 했으니 그냥 sentence[0]과 sentence[sentence.length()-1] 위치가 공백인지 확인하고 제거해주는 게 더 간단하겠지만,

erase 함수를 이참에 써보면 좋지 않을까?

​

스트링.erase(시작 인덱스, 끝 인덱스)

-> 시작 인덱스~끝 인덱스-1까지를 삭제한다.(끝 인덱스는 포함되지 않는다.)

스트링.erase(인덱스)

->인덱스~끝까지를 삭제한다.

​

find_first_not_of(' ')는 앞에서부터 ' ', 즉 공백이 아닌 첫번째 문자 위치를 리턴한다. 그걸 erase의 긑 인덱스로 사용하면, 끝 인덱스는 포함되지 않으므로 바로 앞의 문자(=공백)까지만 삭제된다.

find_last_not_of(' ')는 뒤에서부터 공백이 아닌 첫번째 문자 위치를 리턴한다. erase의 시작 위치 인덱스는 삭제할 때 포함되기 때문에, +1을 해줘서 공백 위치부터 삭제되도록 해준다.

​

(2) std::getline(iss, word, ' ')

스트림에서 딜리미터를 기준으로 단어를 읽을 때 쓴다. 리턴값은 읽은 후의 스트림인데, if문에 들어가면 false가 되기 때문에 if(std::getline(iss, word, ' ')) 형태로 사용했다.

​

​

​

*참고로 여기서 처음엔 if(iss.eof()) 를 사용해서 eof를 체크했었는데...테스트 입력은 제대로 세어졌으나 어째선지 제출하면 틀렸다고 나왔다.

eof()는 파일의 끝인데 읽으려 시도해서 실패했을 때 true가 되고,

스트림은 어떤 이유에서든 읽기를 실패했을 때 true가 된다. 즉, eof를 포함하는 개념이다.

답안 제출 후 돌아가는 테스트 과정에서, eof가 아닌 다른 이유로 읽기를 실패하는 경우가 생기는 게 아닐까? 그래서 eof로 조건문을 돌리면 true가 되어 넘어가는 바람에 count가 +1 되고, 스트림을 조건문에 넣으면 제대로 false가 되어 count가 정상적으로 세어진다든가...

​

eof()와 stream의 차이를 아무리 검색해봤지만 이정도밖에 나오지 않음.

이유는 모르겠지만 이럴 때 아무튼 eof 사용하지 말고 stream을 if문에 넣어서 체크해야 할 것 같다.
