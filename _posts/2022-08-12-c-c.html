﻿char c = (char)30;

간단하게 말해 위 코드는 C 스타일의 형 변환 연산이다.

괄호 안에 형 변환할 타입을 넣어주기만 하면 부모 타입에서 자식 타입으로, 자식 타입에서 부모 타입으로, 전혀 상관없는 타입끼리의 변환에도 마구 쓸 수 있다.

​

지금까진 이 방법을 그냥 써왔는데(어제까지도),

공부를 하다보면 프로그래밍 문법은 이런 사례를 점점 없애가려 하는 걸 알 수 있다.

전적으로 프로그래머의 역량에 맡겼던 과거와 달리 지금은 문법 차원에서 차단하는 걸 선호하는 듯하다.

따라서 얘도 이제는 쓰지 않는 문법인 것 같다.(권장되지 않는, 위험한 코드라고 한다)

​

C++에서는 이를 대체하는 형 변환으로 네 가지 함수를 제공한다.

dynamic_cast, static_cast, const_cast, 그리고 reinterpret_cast이다.



1. dynamic_cast

자식 클래스의 포인터 및 참조형을 부모 클래스의 포인터 및 참조형으로 형 변환 하려는 경우

(참조형과 포인터에 대한 포스트는 여기 ↓)

https://sealbinary.blogspot.com/2022/08/c-const.html

dynamic_cast의 사용법은 다음과 같다.

dynamic_cast<T>(value)

예를 들면, Human과 Student라는 클래스가 있다고 했을 때

class Human { ... } class Student : Human { ... } int main(){ Student *pStudent = new Student(); Human *pHuman = dynamic_cast<Human*>(pStudent); }

이런 식으로 쓴다.

반드시 자식 클래스의 포인터 -> 부모 클래스의 포인터로 형 변환만 가능하다. 반대의 경우 컴파일 에러가 난다.

​

이 연산은 네 가지 캐스팅 중에서 가장 안전한 연산이다.

[부모 -> 자식 포인터로의 변환이 안전하지 않은 이유]

가령, Student 클래스에는 소속 학교나 학년, 성적 등의 멤버변수가 포함되어 있겠지만 Human에는 그런 것이 없다. 이때 Student의 포인터를 Human의 포인터로 캐스팅하면 Human을 다룰 때 필요한 모든 정보는 다 제대로 존재하기 때문에 문제가 발생하지 않는다. 하지만 반대로 Human의 포인터를 Student의 포인터로 캐스팅을 허용하게 되면, 소속 학교, 학년, 성적 항목이 존재하지 않는 상태로 Student 타입으로 다루게 되고, 예상치 못한 결과가 일어난다.

부모 -> 자식 포인터로의 연산은 dynamic_cast로는 할 수 없고 static_cast로 해야 한다.

(*단, 예외적으로 부모 클래스에 가상 함수가 있을 경우엔 부모->자식 클래스로의 형 변환을 해도 컴파일 에러가 나지 않는다. 하지만 캐스팅에 실패할 경우 null을 리턴한다.)

(*캐스팅에 성공하는 경우는 다음과 같다.)

Human *pHuman = new Student(); Student *pStudent = dynamic_cast<Student>(pHuman);

(포인터 변수의 참조타입은 부모 클래스이지만 실제 참조하는 값은 자식 클래스일 때)

2. static_cast

부모 클래스의 포인터 및 참조형을 자식 클래스의 포인터 및 참조형으로 형 변환 하려는 경우.

또는 기본 자료형 간의 캐스팅

부모->자식 클래스로의 형 변환이 필요할 땐 이 캐스팅을 사용한다.

또한, int를 double로 바꾸거나 double을 int로 바꾸는 등의 캐스팅에도 이 캐스팅을 사용한다.

엥간한 캐스팅은 이 함수로 할 수 있다.

​

다만 이 캐스팅 함수로 할 수 없는 캐스팅은 "const 변수를 const 타입을 참조하는 포인터가 아닌 포인터로 초기화하는 것"이다.

int const a = 10; int* p = &a; //컴파일 에러! int* p = static_cast(&a); //컴파일 에러!

위의 코드에서 a는 const int 변수이다. 즉, a의 값을 절대 변경할 수 없다.

따라서 포인터로 a를 가리키면 *p = 11 같은 식으로 값을 변경할 수 있게 되기 때문에 일반 포인터엔 담을 수 없고 타입을 맞춰서

const int* p = static_cast<const int*>(&a);

위처럼 해야 한다. (사실 그냥 const int* p = &a 해도 되긴 한다.)

​

참고로, c 스타일의 캐스팅은 기묘하게도 저게 가능하다고 한다.

int* p = (int*) &a; *p = 11;

이러면 진짜 a의 값이 바뀐다. const int인데도.

아무튼 static_cast도 다 풀어준 것 같지만 저런 기묘한 캐스팅은 막혀있으므로 c 스타일의 캐스팅보다 훨씬 안전하다고 한다.

​

​

3. const_cast

const 성향을 제거하는 형 변환 연산자

바로 위에서 const를 포인터로 참조해 바꾸는 게 기묘한 연산이라 막아뒀다고 했지만, 이게 필요한 경우가 있는 듯하다.

가령, 함수의 파라미터로 보내야 하는데 이 함수의 파라미터 타입이 const가 아닐 때, const_cast로 const 한정자를 제거하고 넘겨주면 정상 실행이 가능해진다.

사실 함수 파라미터 타입도 const로 변경해주는 게 베스트겠지만 함수를 마음대로 변경할 수 없는 상황에서 유용할 것 같다.

void func(char* str){ } int main(){ const char* str = "abc"; func(const_cast<char*>(str)); }

이렇게 쓴다. 참고로 c에 string 타입이 없어서 char* 타입으로 문자열을 다룬 건 아주 익숙한 방식이지만 C11부터는 char* 변수에 문자열을 초기화할 때 반드시 const char* 타입으로만 가능하도록 바뀌었다고 한다.

사실 "abc"는 문자 리터럴이라 값을 변경할 수 없고, const char* 타입인데 과거의 c에선 char*형 변수에 집어넣을 때 암시적 캐스팅이 일어났던 거라고 한다. C11부턴 암시적 형변환이 일어나지 않는 것이고.

따라서 const_cast는 습관적으로 문자 리터럴을 char*로 다루려 할 때 유용하지 않을까 싶다.

​

물론 위험한 캐스팅인 것도 맞다.

​

​

4. reinterpret_cast

포인터를 대상으로 한 모든 유형의 형 변환을 허용.

제일 위험한 캐스팅.

이지만 유용한 경우가 있긴 한듯.

​

가령, int* 포인터는 int의 크기인 4바이트 단위로 값에 접근을 하는데, 이를 char* 포인터로 형 변환한 후 접근하면 바이트 단위로 접근할 수 있게 된다.

​

포인터<->정수 간의 연산도 가능해서, 주소값을 정수로 확인할 때에도 쓰인다.

​
